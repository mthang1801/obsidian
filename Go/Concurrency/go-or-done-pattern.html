Or-Done Pattern

Kết hợp Context với Channel Cleanup

Or-Done Pattern (còn gọi là orDone) là một trong những pattern thiết yếu trong lập trình concurrency với Go — cho phép thoát an toàn khỏi goroutine bằng cách lắng nghe đồng thời tín hiệu từ context.Done() và từ channel dữ liệu, đảm bảo không xảy ra goroutine leak hay deadlock. Tài liệu này đi từ định nghĩa cơ bản đến kỹ thuật expert-level.

01. Or-Done Pattern là gì?

Or-Done Pattern xuất phát từ cuốn sách nổi tiếng "Concurrency in Go" của Katherine Cox-Buday (O'Reilly). Pattern này giải quyết một vấn đề cực kỳ phổ biến: làm thế nào để đọc dữ liệu từ một channel nhưng vẫn có thể thoát sớm khi context bị hủy, timeout xảy ra, hoặc một điều kiện dừng nào đó được kích hoạt.

📌 Định nghĩa chính thức Or-Done Pattern là cách bọc một channel đọc bình thường (for v := range ch) bên trong một goroutine có tích hợp select, cho phép thoát sớm khi nhận được tín hiệu từ done channel hoặc ctx.Done() — mà không block vô hạn.

Xét vấn đề cơ bản: khi bạn cần đọc từ channel nhưng goroutine producer đã bị hủy:

❌ vấn_de.go — GOROUTINE LEAK ANTI-PATTERN 
// ❌ Vấn đề: nếu ctx bị cancel, vòng lặp này sẽ BLOCK mãi mãi
// vì ch không bao giờ được gửi thêm dữ liệu hoặc đóng.
func badConsumer(ctx context.Context, ch <-chan string) {
    for v := range ch {  // ← BỊ BLOCK nếu ch không có data và không đóng
        fmt.Println(v)
    }
    // ctx.Done() KHÔNG BAO GIỜ được kiểm tra → goroutine leak!
}
✅ or_done.go — GIẢI PHÁP CORRECT 
// ✅ OrDone bọc channel đọc với khả năng cancel sớm
func OrDone(ctx context.Context, ch <-chan interface{}) <-chan interface{} {
    valStream := make(chan interface{})
    go func() {
        defer close(valStream)
        for {
            select {
            case <-ctx.Done():   // Tín hiệu 1: context bị hủy
                return
            case v, ok := <-ch: // Tín hiệu 2: nhận dữ liệu hoặc channel đóng
                if !ok {
                    return       // Channel đã đóng → thoát
                }
                select {
                case <-ctx.Done():
                case valStream <- v: // Forward giá trị
                }
            }
        }
    }()
    return valStream
}

02. Tại sao Or-Done Pattern quan trọng?

❌ Không dùng Or-Done

  • Goroutine bị block vô hạn
  • Context cancel không có tác dụng
  • Memory leak tích lũy theo thời gian
  • Không thể graceful shutdown
  • Deadlock khi producer đã dừng

✅ Dùng Or-Done

  • Goroutine thoát sạch khi cancel
  • Timeout và deadline hoạt động đúng
  • Không memory leak
  • Pipeline có thể dừng ở bất kỳ tầng nào
  • Composable với các pattern khác
Tình huống Không có Or-Done Có Or-Done
Context timeout sau 5s Goroutine vẫn chạy Goroutine thoát ngay
Producer bị crash Consumer block mãi Consumer detect & thoát
HTTP request bị cancel Pipeline vẫn chạy ngầm Toàn bộ pipeline dừng
Signal SIGTERM Process không thoát được Graceful shutdown
Test với t.Cleanup Test goroutine leak Clean test teardown

04. Cài đặt cơ bản

Đây là phiên bản đơn giản nhất của Or-Done Pattern — dùng interface{} (Go < 1.18):

📄 or_done_basic.go CƠ BẢN 
package main

import (
    "context"
    "fmt"
    "time"
)

// OrDone bọc ch để đảm bảo goroutine thoát khi:
// 1. ctx bị cancel/timeout
// 2. ch bị đóng (channel closed)
func OrDone(ctx context.Context, ch <-chan interface{}) <-chan interface{} {
    valStream := make(chan interface{})
    go func() {
        defer close(valStream) // Đảm bảo consumer luôn nhận được tín hiệu đóng
        for {
            select {
            case <-ctx.Done():
                return
            case v, ok := <-ch:
                if !ok {
                    return // Channel đã đóng
                }
                // Inner select: tránh block khi gửi vào valStream
                select {
                case <-ctx.Done():
                    return
                case valStream <- v:
                }
            }
        }
    }()
    return valStream
}

func main() {
    // Tạo context với timeout 2 giây
    ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
    defer cancel()

    // Producer: gửi số mỗi 500ms
    ch := make(chan interface{})
    go func() {
        defer close(ch)
        for i := 0; ; i++ {
            select {
            case <-ctx.Done():
                return
            case ch <- i:
                time.Sleep(500 * time.Millisecond)
            }
        }
    }()

    // Consumer: đọc qua OrDone — sẽ tự thoát sau 2 giây
    for v := range OrDone(ctx, ch) {
        fmt.Printf("Nhận được: %v\n", v)
    }
    fmt.Println("Consumer đã thoát sạch ✓")
}

// Output:
// Nhận được: 0
// Nhận được: 1
// Nhận được: 2
// Nhận được: 3
// Consumer đã thoát sạch ✓
💡 Tại sao cần Inner select? Inner select (select thứ hai bên trong) ngăn chặn trường hợp: goroutine đã nhận được dữ liệu từ ch, nhưng khi gửi vào valStream lại bị block vì consumer chưa đọc. Nếu lúc đó ctx bị cancel, không có inner select, goroutine sẽ stuck ở đây.

05. Tích hợp Context — Cancellation & Deadline

Minh họa đầy đủ ba loại context thường gặp kết hợp với Or-Done:

📄 context_integration.go NÂNG CAO 
package main

import (
    "context"
    "fmt"
    "log"
    "time"
)

// ─── 1. WithCancel: Cancel thủ công ──────────────────────────────
func demoWithCancel() {
    ctx, cancel := context.WithCancel(context.Background())
    ch := infiniteNumbers(ctx)

    // Cancel sau khi đọc được 5 số
    count := 0
    for v := range OrDone(ctx, ch) {
        fmt.Printf("[Cancel] %v\n", v)
        count++
        if count >= 5 {
            cancel() // Kích hoạt cancel
        }
    }
    fmt.Println("[Cancel] Xong ✓")
}

// ─── 2. WithTimeout: Tự động timeout ─────────────────────────────
func demoWithTimeout() {
    ctx, cancel := context.WithTimeout(context.Background(), 1500*time.Millisecond)
    defer cancel()

    ch := slowProducer(ctx, 300*time.Millisecond)

    for v := range OrDone(ctx, ch) {
        fmt.Printf("[Timeout] Nhận: %v\n", v)
    }
    if ctx.Err() == context.DeadlineExceeded {
        fmt.Println("[Timeout] Context deadline vượt quá ✓")
    }
}

// ─── 3. WithDeadline: Thời điểm cụ thể ──────────────────────────
func demoWithDeadline() {
    deadline := time.Now().Add(2 * time.Second)
    ctx, cancel := context.WithDeadline(context.Background(), deadline)
    defer cancel()

    ch := infiniteNumbers(ctx)

    for v := range OrDone(ctx, ch) {
        fmt.Printf("[Deadline] %v\n", v)
        time.Sleep(600 * time.Millisecond)
    }
    log.Printf("Dừng lúc: %v, Lý do: %v\n", time.Now().Format("15:04:05"), ctx.Err())
}

// ─── Helpers ──────────────────────────────────────────────────────
func infiniteNumbers(ctx context.Context) <-chan interface{} {
    ch := make(chan interface{})
    go func() {
        defer close(ch)
        for i := 0; ; i++ {
            select {
            case <-ctx.Done():
                return
            case ch <- i:
            }
        }
    }()
    return ch
}

func slowProducer(ctx context.Context, delay time.Duration) <-chan interface{} {
    ch := make(chan interface{})
    go func() {
        defer close(ch)
        for i := 0; ; i++ {
            time.Sleep(delay)
            select {
            case <-ctx.Done():
                return
            case ch <- fmt.Sprintf("item-%d", i):
            }
        }
    }()
    return ch
}

func main() {
    fmt.Println("=== Demo 1: WithCancel ===")
    demoWithCancel()

    fmt.Println("\n=== Demo 2: WithTimeout ===")
    demoWithTimeout()

    fmt.Println("\n=== Demo 3: WithDeadline ===")
    demoWithDeadline()
}

06. Phiên bản Generic — Go 1.18+

Từ Go 1.18, chúng ta có thể tạo OrDone type-safe với generics, tránh cần type assertion:

📄 or_done_generic.go Go 1.18+ 
package concurrency

import "context"

// OrDone[T] — phiên bản generic, type-safe
// T là kiểu dữ liệu của channel
func OrDone[T any](ctx context.Context, ch <-chan T) <-chan T {
    valStream := make(chan T)
    go func() {
        defer close(valStream)
        for {
            select {
            case <-ctx.Done():
                return
            case v, ok := <-ch:
                if !ok {
                    return
                }
                select {
                case <-ctx.Done():
                    return
                case valStream <- v:
                }
            }
        }
    }()
    return valStream
}

// Sử dụng với nhiều kiểu khác nhau:

type Event struct {
    ID      string
    Payload []byte
}

func processEvents(ctx context.Context, events <-chan Event) {
    // Không cần type assertion — type-safe hoàn toàn
    for e := range OrDone(ctx, events) {
        fmt.Printf("Event: %s\n", e.ID) // e đã là type Event
    }
}

func processStrings(ctx context.Context, strs <-chan string) {
    for s := range OrDone(ctx, strs) {
        fmt.Println(s) // s là string trực tiếp
    }
}

func processInts(ctx context.Context, nums <-chan int) {
    sum := 0
    for n := range OrDone(ctx, nums) {
        sum += n // n là int trực tiếp
    }
}

07. Multi-channel OrDone — Merge nhiều nguồn

📄 multi_or_done.go NÂNG CAO 
package main

import (
    "context"
    "fmt"
    "sync"
)

// MergeOrDone: merge nhiều channel thành một, với cancel support
// Khi ctx bị cancel → tất cả goroutine đọc đều thoát
func MergeOrDone[T any](ctx context.Context, channels ...<-chan T) <-chan T {
    out := make(chan T)
    var wg sync.WaitGroup

    // Mỗi channel có một goroutine đọc riêng
    multiplex := func(ch <-chan T) {
        defer wg.Done()
        for v := range OrDone(ctx, ch) { // OrDone đảm bảo thoát khi cancel
            select {
            case <-ctx.Done():
                return
            case out <- v:
            }
        }
    }

    wg.Add(len(channels))
    for _, ch := range channels {
        go multiplex(ch)
    }

    // Đóng out khi tất cả goroutine đã xong
    go func() {
        wg.Wait()
        close(out)
    }()

    return out
}

func main() {
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()

    // Tạo 3 nguồn dữ liệu
    ch1 := makeSource(ctx, "A", 100)
    ch2 := makeSource(ctx, "B", 200)
    ch3 := makeSource(ctx, "C", 150)

    count := 0
    for v := range MergeOrDone(ctx, ch1, ch2, ch3) {
        fmt.Println(v)
        count++
        if count == 10 {
            cancel() // Dừng toàn bộ sau 10 items
        }
    }
    fmt.Printf("Tổng nhận: %d items\n", count)
}

func makeSource(ctx context.Context, prefix string, delayMs int) <-chan string {
    ch := make(chan string)
    go func() {
        defer close(ch)
        for i := 0; ; i++ {
            select {
            case <-ctx.Done():
                return
            case ch <- fmt.Sprintf("%s-%d", prefix, i):
                time.Sleep(time.Duration(delayMs) * time.Millisecond)
            }
        }
    }()
    return ch
}

08. Nâng cao — Pipeline tích hợp Or-Done

Or-Done thực sự tỏa sáng khi kết hợp với concurrent pipeline nhiều tầng:

🔗 Pipeline với Or-Done ở mỗi tầng
📄 pipeline_or_done.go NÂNG CAO 
package main

import (
    "context"
    "fmt"
    "strings"
)

// Pipeline 3 tầng: Generate → Filter → Transform → Consume
// Mỗi tầng dùng OrDone để đảm bảo thoát sạch

func Generate(ctx context.Context, values ...string) <-chan string {
    ch := make(chan string)
    go func() {
        defer close(ch)
        for _, v := range values {
            select {
            case <-ctx.Done():
                return
            case ch <- v:
            }
        }
    }()
    return ch
}

// Tầng 1: Filter — chỉ giữ lại string có độ dài > 3
func Filter(ctx context.Context, in <-chan string) <-chan string {
    out := make(chan string)
    go func() {
        defer close(out)
        for v := range OrDone(ctx, in) { // ← OrDone bảo vệ tầng này
            if len(v) > 3 {
                select {
                case <-ctx.Done():
                    return
                case out <- v:
                }
            }
        }
    }()
    return out
}

// Tầng 2: Transform — uppercase
func ToUpper(ctx context.Context, in <-chan string) <-chan string {
    out := make(chan string)
    go func() {
        defer close(out)
        for v := range OrDone(ctx, in) { // ← OrDone bảo vệ tầng này
            select {
            case <-ctx.Done():
                return
            case out <- strings.ToUpper(v):
            }
        }
    }()
    return out
}

func main() {
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()

    words := []string{"go", "golang", "hi", "context", "ok", "channel"}

    // Kết nối pipeline: Generate → Filter → ToUpper → Consumer
    src     := Generate(ctx, words...)
    filtered := Filter(ctx, src)
    upper   := ToUpper(ctx, filtered)

    for result := range OrDone(ctx, upper) { // ← Consumer cũng dùng OrDone
        fmt.Println(result)
    }
}

// Output:
// GOLANG
// CONTEXT
// CHANNEL

Fan-Out với Channel Cleanup

📄 fanout_cleanup.go FAN-OUT 
package main

import (
    "context"
    "fmt"
    "sync"
)

// FanOut: phân phối 1 channel nguồn thành N worker channel
// Mỗi worker được bọc bởi OrDone → cleanup đồng loạt khi cancel
func FanOut[T any](ctx context.Context, in <-chan T, numWorkers int) []<-chan T {
    workers := make([]<-chan T, numWorkers)
    for i := range workers {
        workers[i] = workerProcess(ctx, OrDone(ctx, in), i)
    }
    return workers
}

func workerProcess[T any](ctx context.Context, in <-chan T, id int) <-chan T {
    out := make(chan T)
    go func() {
        defer close(out)
        for v := range in {
            fmt.Printf("Worker[%d] xử lý: %v\n", id, v)
            select {
            case <-ctx.Done():
                return
            case out <- v:
            }
        }
    }()
    return out
}

// FanIn: gom kết quả từ nhiều worker về 1 channel
func FanIn[T any](ctx context.Context, workers []<-chan T) <-chan T {
    out := make(chan T)
    var wg sync.WaitGroup

    for _, w := range workers {
        wg.Add(1)
        go func(ch <-chan T) {
            defer wg.Done()
            for v := range OrDone(ctx, ch) {
                select {
                case <-ctx.Done():
                    return
                case out <- v:
                }
            }
        }(w)
    }

    go func() { wg.Wait(); close(out) }()
    return out
}

Timeout Chain Pattern

📄 timeout_chain.go TIMEOUT 
package main

import (
    "context"
    "fmt"
    "time"
)

// OrDoneWithTimeout: kết hợp timeout per-item với context cancel
// Nếu không nhận được item trong `itemTimeout`, tự động thoát
func OrDoneWithTimeout[T any](
    ctx context.Context,
    ch <-chan T,
    itemTimeout time.Duration,
) <-chan T {
    out := make(chan T)
    go func() {
        defer close(out)
        for {
            timer := time.NewTimer(itemTimeout)
            select {
            case <-ctx.Done():
                timer.Stop()
                return
            case <-timer.C:
                fmt.Println("⏱️  Item timeout — không có dữ liệu")
                return
            case v, ok := <-ch:
                timer.Stop()
                if !ok {
                    return
                }
                select {
                case <-ctx.Done():
                    return
                case out <- v:
                }
            }
        }
    }()
    return out
}

func main() {
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()

    ch := make(chan int)
    go func() {
        defer close(ch)
        for i := 0; i < 5; i++ {
            ch <- i
            if i == 2 {
                time.Sleep(3 * time.Second) // Giả lập delay lớn
            }
        }
    }()

    // Timeout 1 giây nếu không có item mới
    for v := range OrDoneWithTimeout(ctx, ch, 1*time.Second) {
        fmt.Printf("Nhận: %d\n", v)
    }

    // Output:
    // Nhận: 0
    // Nhận: 1
    // Nhận: 2
    // ⏱️  Item timeout — không có dữ liệu
}

ErrGroup Hybrid Pattern

📄 errgroup_hybrid.go ERRGROUP 
package main

import (
    "context"
    "fmt"
    "golang.org/x/sync/errgroup"
)

type Result struct {
    Source string
    Value  int
}

// ErrGroupOrDone: kết hợp errgroup + OrDone
// Khi bất kỳ goroutine nào fail → ctx bị cancel → OrDone cleanup tất cả
func ErrGroupOrDone(ctx context.Context, sources []string) (<-chan Result, func() error) {
    g, gCtx := errgroup.WithContext(ctx)
    results := make(chan Result, len(sources))

    for _, src := range sources {
        src := src
        g.Go(func() error {
            // Giả lập fetch từ source
            val, err := fetchFromSource(gCtx, src)
            if err != nil {
                return fmt.Errorf("source %s: %w", src, err)
            }
            select {
            case <-gCtx.Done():
                return gCtx.Err()
            case results <- Result{Source: src, Value: val}:
            }
            return nil
        })
    }

    go func() {
        _ = g.Wait()
        close(results) // Đóng khi tất cả goroutine xong hoặc fail
    }()

    // Trả về channel đã được OrDone bảo vệ
    return OrDone(ctx, results), g.Wait
}

func fetchFromSource(ctx context.Context, src string) (int, error) {
    // Simulate fetch...
    select {
    case <-ctx.Done():
        return 0, ctx.Err()
    default:
        return len(src) * 10, nil
    }
}

func main() {
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()

    ch, wait := ErrGroupOrDone(ctx, []string{"db", "cache", "api"})
    for r := range ch {
        fmt.Printf("Source: %s, Value: %d\n", r.Source, r.Value)
    }
    if err := wait(); err != nil {
        fmt.Printf("Lỗi: %v\n", err)
    }
}

09. Expert — Lock-Free & Reactive Stream

📄 lock_free_or_done.go EXPERT 
package main

import (
    "context"
    "fmt"
    "runtime"
    "sync/atomic"
    "unsafe"
)

// AtomicOrDone: sử dụng atomic để tránh race condition
// trong trường hợp nhiều goroutine cùng close channel
type SafeChannel[T any] struct {
    ch     chan T
    closed uint32 // atomic flag
}

func NewSafeChannel[T any](size int) *SafeChannel[T] {
    return &SafeChannel[T]{ch: make(chan T, size)}
}

func (sc *SafeChannel[T]) SafeClose() {
    if atomic.CompareAndSwapUint32(&sc.closed, 0, 1) {
        close(sc.ch)
    }
}

func (sc *SafeChannel[T]) Send(ctx context.Context, v T) bool {
    if atomic.LoadUint32(&sc.closed) == 1 {
        return false
    }
    select {
    case <-ctx.Done():
        return false
    case sc.ch <- v:
        return true
    }
}

func (sc *SafeChannel[T]) OrDone(ctx context.Context) <-chan T {
    return OrDone(ctx, sc.ch)
}

// ─── Reactive Stream với back-pressure ─────────────────────────────

type ReactiveStream[T any] struct {
    ctx        context.Context
    in         <-chan T
    bufferSize int
}

func NewReactiveStream[T any](ctx context.Context, in <-chan T, bufferSize int) *ReactiveStream[T] {
    return &ReactiveStream[T]{ctx: ctx, in: in, bufferSize: bufferSize}
}

// Map: transform mỗi item
func (rs *ReactiveStream[T]) Map(fn func(T) T) *ReactiveStream[T] {
    out := make(chan T, rs.bufferSize)
    go func() {
        defer close(out)
        for v := range OrDone(rs.ctx, rs.in) {
            select {
            case <-rs.ctx.Done():
                return
            case out <- fn(v):
            }
        }
    }()
    return NewReactiveStream(rs.ctx, out, rs.bufferSize)
}

// Filter: lọc item
func (rs *ReactiveStream[T]) Filter(pred func(T) bool) *ReactiveStream[T] {
    out := make(chan T, rs.bufferSize)
    go func() {
        defer close(out)
        for v := range OrDone(rs.ctx, rs.in) {
            if !pred(v) {
                continue
            }
            select {
            case <-rs.ctx.Done():
                return
            case out <- v:
            }
        }
    }()
    return NewReactiveStream(rs.ctx, out, rs.bufferSize)
}

// Take: chỉ lấy N item đầu tiên rồi cancel
func (rs *ReactiveStream[T]) Take(n int) *ReactiveStream[T] {
    out := make(chan T, rs.bufferSize)
    go func() {
        defer close(out)
        for i := 0; i < n; i++ {
            select {
            case <-rs.ctx.Done():
                return
            case v, ok := <-OrDone(rs.ctx, rs.in):
                if !ok {
                    return
                }
                out <- v
            }
        }
    }()
    return NewReactiveStream(rs.ctx, out, rs.bufferSize)
}

// Subscribe: terminal operation
func (rs *ReactiveStream[T]) Subscribe(handler func(T)) {
    for v := range OrDone(rs.ctx, rs.in) {
        handler(v)
    }
}

// Sử dụng:
func main() {
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()

    nums := infiniteInts(ctx)

    NewReactiveStream(ctx, nums, 16).
        Filter(func(n int) bool { return n%2 == 0 }).  // chỉ số chẵn
        Map(func(n int) int { return n * n }).         // bình phương
        Take(5).                                          // lấy 5 cái
        Subscribe(func(n int) { fmt.Println(n) })

    // Output: 0, 4, 16, 36, 64
}

func infiniteInts(ctx context.Context) <-chan int {
    ch := make(chan int)
    go func() {
        defer close(ch)
        for i := 0; ; i++ {
            select {
            case <-ctx.Done(): return
            case ch <- i:
            }
        }
    }()
    return ch
}

package main

import (
    "context"
    "fmt"
    "log/slog"
    "sync/atomic"
    "time"
)

type OrDoneMetrics struct {
    Received    int64
    Forwarded   int64
    Dropped     int64
    StartTime   time.Time
}

// OrDoneInstrumented: phiên bản có metrics & tracing
func OrDoneInstrumented[T any](
    ctx context.Context,
    ch <-chan T,
    name string,
) (<-chan T, *OrDoneMetrics) {
    out := make(chan T)
    m := &OrDoneMetrics{StartTime: time.Now()}

    go func() {
        defer close(out)
        defer func() {
            slog.Info("OrDone selesai",
                "name", name,
                "received", atomic.LoadInt64(&m.Received),
                "forwarded", atomic.LoadInt64(&m.Forwarded),
                "dropped", atomic.LoadInt64(&m.Dropped),
                "duration", time.Since(m.StartTime),
            )
        }()

        for {
            select {
            case <-ctx.Done():
                slog.Debug("OrDone: ctx cancelled", "name", name)
                return
            case v, ok := <-ch:
                if !ok {
                    slog.Debug("OrDone: channel closed", "name", name)
                    return
                }
                atomic.AddInt64(&m.Received, 1)
                select {
                case <-ctx.Done():
                    atomic.AddInt64(&m.Dropped, 1)
                    return
                case out <- v:
                    atomic.AddInt64(&m.Forwarded, 1)
                }
            }
        }
    }()

    return out, m
}

func main() {
    ctx, cancel := context.WithTimeout(context.Background(), 500*time.Millisecond)
    defer cancel()

    ch := infiniteInts(ctx)
    traced, metrics := OrDoneInstrumented(ctx, ch, "main-stream")

    for v := range traced {
        _ = v
    }

    fmt.Printf("Metrics: received=%d forwarded=%d\n",
        atomic.LoadInt64(&metrics.Received),
        atomic.LoadInt64(&metrics.Forwarded),
    )
}

11. Best Practices — Bảng tổng hợp

1

Luôn dùng Or-Done khi đọc channel trong goroutine có context

Nếu hàm nhận context.Context và cần đọc từ channel, đây là dấu hiệu cần OrDone. Không bao giờ dùng for v := range ch thuần khi ctx có thể bị cancel.

2

Luôn có inner select khi forward dữ liệu

Sau khi nhận được giá trị từ ch, việc gửi vào out cũng cần được bao bởi select { case <-ctx.Done() } để tránh block.

3

Dùng generic ([T any]) khi dùng Go 1.18+

Tránh interface{} và type assertion. Generic giúp code type-safe, ít lỗi runtime, IDE hỗ trợ đầy đủ, và không có boxing overhead.

4

Kết hợp goleak trong tests

Thêm defer goleak.VerifyNone(t) vào đầu mỗi test để tự động phát hiện goroutine leak nếu OrDone không thoát đúng cách.

5

Buffer channel khi throughput cao

Nếu producer gửi nhanh hơn consumer xử lý, dùng buffered channel (ví dụ: make(chan T, 64)) trong OrDone để tránh producer bị block quá nhiều.

⚠️ Anti-Pattern: Đừng share OrDone output giữa nhiều consumer Kết quả trả về từ OrDone là một channel — mỗi item chỉ được đọc bởi một goroutine. Nếu cần nhiều consumer, dùng MergeOrDone hoặc kết hợp với Fan-Out pattern.
