01 — Tổng Quan về Channel

Channel là cơ chế giao tiếp cốt lõi trong Go, được xây dựng dựa trên mô hình CSP (Communicating Sequential Processes) của Tony Hoare. Thay vì chia sẻ bộ nhớ giữa các goroutines, Go khuyến khích việc chia sẻ dữ liệu thông qua channel.

💡

Go Concurrency Mantra

"Don't communicate by sharing memory; share memory by communicating." — Rob Pike

Mô Hình CSP — Channel như một "Pipe" giữa các Goroutines

Channel là gì?

Channel trong Go là một typed conduit (ống dẫn có kiểu dữ liệu) cho phép các goroutines gửi và nhận giá trị. Channel đảm bảo thread-safety mà không cần mutex thủ công.

📌 Khai báo Channel

// Unbuffered channel
ch := make(chan int)

// Buffered channel (capacity=5)
bch := make(chan int, 5)

// Directional channels
var send chan<- int  // send-only
var recv <-chan int  // receive-only

📌 Operations cơ bản

// Gửi giá trị vào channel
ch <- 42

// Nhận giá trị từ channel
v := <-ch

// Nhận + kiểm tra channel đã đóng
v, ok := <-ch

// Đóng channel (chỉ sender đóng)
close(ch)
Synchronous Communication

02 — Unbuffered Channel

Unbuffered channel (hay synchronous channel) là loại channel không có bộ đệm. Mỗi lần gửi sẽ block goroutine gửi cho đến khi có goroutine khác sẵn sàng nhận, và ngược lại.

make(chan T)  ≡  make(chan T, 0)   |   capacity = 0, len = 0
Cơ Chế Hoạt Động — Unbuffered Channel (Rendezvous)

Khi nào Unbuffered Channel block?

🔒

Sender BLOCK khi:

Không có goroutine nào đang chờ nhận (<-ch). Sender sẽ bị treo cho đến khi receiver sẵn sàng.

🔒

Receiver BLOCK khi:

Không có goroutine nào đang gửi. Receiver sẽ bị treo cho đến khi sender gửi giá trị hoặc channel bị đóng.

Code Example — Unbuffered Channel

unbuffered_basic.go Go 
package main

import (
    "fmt"
    "time"
)

func main() {
    // Tạo unbuffered channel — capacity = 0
    ch := make(chan string)

    // Goroutine sender
    go func() {
        fmt.Println("[Sender] Chuẩn bị gửi...")
        // Dòng này BLOCK cho đến khi receiver sẵn sàng nhận
        ch <- "Hello, Goroutine!"
        fmt.Println("[Sender] Đã gửi xong, tiếp tục...")
    }()

    // Giả lập main goroutine làm việc trước
    time.Sleep(500 * time.Millisecond)

    fmt.Println("[Main] Bắt đầu nhận...")
    // Tại đây, receiver sẵn sàng → handoff xảy ra
    msg := <-ch
    fmt.Printf("[Main] Nhận được: %s\n", msg)
}

// Output:
// [Sender] Chuẩn bị gửi...
// (500ms trôi qua — sender đang blocked)
// [Main] Bắt đầu nhận...
// [Sender] Đã gửi xong, tiếp tục...
// [Main] Nhận được: Hello, Goroutine!

Ứng dụng thực tế: Synchronization Primitive

unbuffered_sync.go Go 
package main

import (
    "fmt"
    "sync"
)

// Sử dụng unbuffered channel như một "done signal"
func processItems(items []int, done chan<- bool) {
    var wg sync.WaitGroup
    results := make(chan int, len(items))

    for _, item := range items {
        wg.Add(1)
        go func(n int) {
            defer wg.Done()
            results <- n * n // tính bình phương
        }(item)
    }

    go func() {
        wg.Wait()
        close(results)
    }()

    sum := 0
    for r := range results {
        sum += r
    }
    fmt.Printf("Tổng bình phương: %d\n", sum)

    // Signal hoàn thành qua unbuffered channel
    // Goroutine này BLOCK cho đến khi main nhận
    done <- true
}

func main() {
    done := make(chan bool) // unbuffered!
    go processItems([]int{1, 2, 3, 4, 5}, done)

    fmt.Println("Main đang chờ kết quả...")
    <-done // Main BLOCK tại đây cho đến khi processItems xong
    fmt.Println("Main tiếp tục sau khi nhận done signal")
}
Asynchronous Communication

03 — Buffered Channel

Buffered channel có một internal queue với dung lượng xác định. Sender có thể gửi dữ liệu vào channel mà không cần block miễn là buffer chưa đầy. Receiver có thể nhận dữ liệu từ buffer bất kể sender có đang chạy hay không.

make(chan T, N)  |  capacity = N  |  len(ch) = số phần tử hiện tại  |  cap(ch) = N
Các Trạng Thái của Buffered Channel (capacity = 3)

Code Example — Buffered Channel

buffered_basic.go Go 
package main

import "fmt"

func main() {
    // Tạo buffered channel với capacity = 3
    ch := make(chan int, 3)

    // Gửi 3 giá trị mà KHÔNG cần goroutine receiver đang chạy
    // Vì buffer còn chỗ trống → KHÔNG BLOCK
    ch <- 10
    ch <- 20
    ch <- 30

    fmt.Printf("len=%d, cap=%d\n", len(ch), cap(ch)) // len=3, cap=3

    // Nhận giá trị (FIFO — lấy ra theo thứ tự gửi vào)
    fmt.Println(<-ch) // 10
    fmt.Println(<-ch) // 20
    fmt.Println(<-ch) // 30

    // Kiểm tra len và cap sau khi nhận hết
    fmt.Printf("len=%d, cap=%d\n", len(ch), cap(ch)) // len=0, cap=3

    // Dòng này sẽ BLOCK vô hạn (buffer trống, không ai gửi)
    // <-ch // deadlock!
}

Buffered Channel làm Semaphore

semaphore_pattern.go Go 
package main

import (
    "fmt"
    "sync"
    "time"
)

// Semaphore — giới hạn số goroutines chạy đồng thời
type Semaphore chan struct{}

func NewSemaphore(maxConcurrent int) Semaphore {
    return make(chan struct{}, maxConcurrent)
}

func (s Semaphore) Acquire() { s <- struct{}{} }
func (s Semaphore) Release() { <-s }

func main() {
    // Chỉ cho phép 3 goroutines chạy đồng thời
    sem := NewSemaphore(3)
    var wg sync.WaitGroup

    for i := 0; i < 10; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()

            sem.Acquire() // Chiếm 1 slot trong buffer
            defer sem.Release() // Trả lại slot khi xong

            fmt.Printf("Worker %d đang chạy\n", id)
            time.Sleep(100 * time.Millisecond)
        }(i)
    }

    wg.Wait()
    fmt.Println("Tất cả workers đã hoàn thành")
}

04 — So Sánh Chi Tiết

So sánh Unbuffered vs Buffered Channel
Tiêu chí 🔵 Unbuffered Channel 🟡 Buffered Channel
Khai báo make(chan T) hoặc make(chan T, 0) make(chan T, N) với N > 0
Capacity 0 (không có buffer) N (số phần tử tối đa có thể chứa)
Send blocking BLOCK ngay — chờ receiver BLOCK chỉ khi buffer đầy
Receive blocking BLOCK ngay — chờ sender BLOCK chỉ khi buffer trống
Synchronization Synchronous — cả 2 gặp nhau Asynchronous — không cần gặp nhau
Guarantee Sender biết chắc receiver đã nhận Sender chỉ biết data đã vào buffer
Memory usage Rất thấp — không cấp phát buffer Tỷ lệ với capacity × sizeof(T)
Throughput Thấp hơn — cần sync mỗi lần Cao hơn — giảm context switches
Use case Signaling, sync primitives, handshake Task queues, pipelines, semaphores
Deadlock risk Cao — dễ bị deadlock nếu sender/receiver không khớp Thấp hơn — buffer có thể hấp thụ burst
len(ch) Luôn là 0 Số phần tử hiện đang trong buffer
cap(ch) Luôn là 0 N (capacity đã khai báo)
Hành Vi Khi Sender Nhanh Hơn Receiver

Pipeline Pattern — End-to-End Example

pipeline.go Go 
package main

import (
    "context"
    "fmt"
    "math"
)

// Stage 1: Generate — tạo số nguyên từ 1 đến n
func generate(ctx context.Context, n int) <-chan int {
    out := make(chan int) // unbuffered
    go func() {
        defer close(out)
        for i := 1; i <= n; i++ {
            select {
            case <-ctx.Done(): return
            case out <- i:
            }
        }
    }()
    return out
}

// Stage 2: Square — tính bình phương
func square(ctx context.Context, in <-chan int) <-chan int {
    out := make(chan int, 5) // buffered để giảm blocking
    go func() {
        defer close(out)
        for n := range in {
            select {
            case <-ctx.Done(): return
            case out <- n * n:
            }
        }
    }()
    return out
}

// Stage 3: Filter — lọc số nguyên tố (bình phương của số nguyên tố)
func filterPrimeSqrt(ctx context.Context, in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for n := range in {
            root := int(math.Sqrt(float64(n)))
            if isPrime(root) {
                select {
                case <-ctx.Done(): return
                case out <- n:
                }
            }
        }
    }()
    return out
}

func isPrime(n int) bool {
    if n < 2 { return false }
    for i := 2; i*i <= n; i++ {
        if n%i == 0 { return false }
    }
    return true
}

func main() {
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()

    // Xây dựng pipeline: generate → square → filter → print
    nums   := generate(ctx, 20)      // 1..20
    squares := square(ctx, nums)   // bình phương
    primes := filterPrimeSqrt(ctx, squares) // lọc

    for n := range primes {
        fmt.Printf("Bình phương của số nguyên tố: %d\n", n)
    }
}
// Output: 4, 9, 25, 49, 121, 169, 289, 361

Worker Pool — Production Pattern

worker_pool.go Go 
package main

import (
    "context"
    "fmt"
    "sync"
    "time"
)

type Job struct {
    ID   int
    Data string
}

type Result struct {
    JobID  int
    Output string
    Err    error
}

// WorkerPool: buffered channel cho job queue + result queue
func WorkerPool(
    ctx context.Context,
    numWorkers int,
    jobs <-chan Job,
) <-chan Result {
    // Buffered result channel — không block workers khi kết quả nhiều
    results := make(chan Result, numWorkers*2)
    var wg sync.WaitGroup

    for i := 0; i < numWorkers; i++ {
        wg.Add(1)
        go func(workerID int) {
            defer wg.Done()
            for {
                select {
                case <-ctx.Done():
                    fmt.Printf("Worker %d: context cancelled\n", workerID)
                    return
                case job, ok := <-jobs:
                    if !ok { return } // channel đã đóng
                    results <- processJob(workerID, job)
                }
            }
        }(i)
    }

    // Goroutine để đóng results khi tất cả workers xong
    go func() {
        wg.Wait()
        close(results)
    }()

    return results
}

func processJob(workerID int, job Job) Result {
    time.Sleep(10 * time.Millisecond) // giả lập xử lý
    return Result{
        JobID:  job.ID,
        Output: fmt.Sprintf("worker%d processed: %s", workerID, job.Data),
    }
}

func main() {
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()

    // Buffered job channel — producer không block ngay
    jobs := make(chan Job, 100)

    go func() {
        defer close(jobs)
        for i := 0; i < 20; i++ {
            jobs <- Job{ID: i, Data: fmt.Sprintf("task-%d", i)}
        }
    }()

    results := WorkerPool(ctx, 5, jobs) // 5 workers

    for r := range results {
        fmt.Printf("Result[%d]: %s\n", r.JobID, r.Output)
    }
}

Select Statement — Multiplexing Channels

select_patterns.go Go 
package main

import (
    "context"
    "fmt"
    "time"
)

func selectPatterns() {
    ch1 := make(chan string, 1)
    ch2 := make(chan string, 1)

    // ===== Pattern 1: Timeout =====
    select {
    case msg := <-ch1:
        fmt.Println("Nhận từ ch1:", msg)
    case <-time.After(1 * time.Second):
        fmt.Println("Timeout! Không nhận được sau 1s")
    }

    // ===== Pattern 2: Non-blocking send/receive =====
    select {
    case ch1 <- "hello":
        fmt.Println("Gửi thành công")
    default:
        // default branch — không block nếu ch1 đầy
        fmt.Println("Channel đầy, bỏ qua")
    }

    // ===== Pattern 3: Multiplexing 2 channels =====
    for {
        select {
        case msg := <-ch1:
            fmt.Println("ch1:", msg)
        case msg := <-ch2:
            fmt.Println("ch2:", msg)
        }
    }
}

// Context cancellation với channel
func longRunningTask(ctx context.Context) error {
    for {
        select {
        case <-ctx.Done():
            // Context bị cancel/timeout — dừng ngay
            return ctx.Err()
        default:
            // Tiếp tục làm việc
            fmt.Println("Đang xử lý...")
            time.Sleep(100 * time.Millisecond)
        }
    }
}

07 — Deadlock & Pitfalls

☠️

Deadlock — Kẻ thù số 1 của Channels

Deadlock xảy ra khi tất cả goroutines đều bị block — không ai có thể tiếp tục. Go runtime sẽ detect và panic với message: fatal error: all goroutines are asleep - deadlock!

Các tình huống Deadlock phổ biến

❌ Deadlock: Gửi không có receiver

deadlock_1.go Go 
func main() {
    ch := make(chan int)
    // ❌ Gửi trong main goroutine
    // Không ai nhận → DEADLOCK!
    ch <- 42
}
✅

Fix:

Gửi trong một goroutine riêng: go func() { ch <- 42 }()

❌ Deadlock: Buffer đầy, không có receiver

deadlock_2.go Go 
func main() {
    ch := make(chan int, 2)
    ch <- 1 // OK
    ch <- 2 // OK
    // ❌ Buffer đầy (cap=2)
    ch <- 3 // DEADLOCK!
}
✅

Fix:

Tăng capacity hoặc có goroutine nhận trước khi buffer đầy.

Goroutine Leak — Deadlock ẩn

goroutine_leak.go Go 
// ❌ Goroutine leak — goroutine bị block mãi mãi
func badExample() {
    results := make(chan string) // unbuffered!

    go func() {
        // Goroutine này sẽ bị block nếu caller thoát sớm
        result, err := doWork()
        if err != nil {
            return // Trả về sớm → goroutine treo mãi tại results <-
        }
        results <- result // ⚠ Nếu không ai nhận → LEAK!
    }()

    select {
    case r := <-results:
        fmt.Println(r)
    case <-time.After(1 * time.Second):
        fmt.Println("Timeout")
        return // ← function thoát, goroutine trên bị leak!
    }
}

// ✅ Fix: Dùng buffered channel hoặc context
func goodExample(ctx context.Context) {
    // Buffered(1) — goroutine có thể gửi ngay cả khi caller không nhận
    results := make(chan string, 1)

    go func() {
        result, err := doWork()
        if err != nil {
            return
        }
        select {
        case results <- result: // gửi nếu có chỗ
        case <-ctx.Done(): // hoặc dừng nếu context cancel
            return
        }
    }()

    select {
    case r := <-results:
        fmt.Println(r)
    case <-ctx.Done():
        fmt.Println("Context cancelled")
    }
}

Gửi vào Closed Channel — Panic!

closed_channel.go Go 
func main() {
    ch := make(chan int, 3)
    close(ch)

    // ❌ PANIC: send on closed channel
    ch <- 42

    // ✅ Nhận từ closed channel (trả về zero value + ok=false)
    v, ok := <-ch
    fmt.Printf("v=%d, ok=%v\n", v, ok) // v=0, ok=false

    // ✅ Range tự dừng khi channel đóng và trống
    ch2 := make(chan int, 3)
    ch2 <- 1; ch2 <- 2; ch2 <- 3
    close(ch2)

    for v := range ch2 {  // tự dừng khi channel closed và empty
        fmt.Println(v) // 1, 2, 3
    }
}
⚡

Quy tắc vàng về Channel

Chỉ người gửi mới được đóng channel. Receiver KHÔNG BAO GIỜ đóng channel. Nếu có nhiều senders, dùng sync.Once hoặc một goroutine coordinator để đóng channel.

08 — Real-world Examples

HTTP Server với Rate Limiting qua Channel

rate_limiter.go Go 
package main

import (
    "fmt"
    "net/http"
    "time"
)

// RateLimiter sử dụng buffered channel + ticker
type RateLimiter struct {
    tokens chan struct{}
}

func NewRateLimiter(rps int) *RateLimiter {
    rl := &RateLimiter{
        // Buffer size = burst size (cho phép burst ngắn hạn)
        tokens: make(chan struct{}, rps),
    }

    // Refill tokens theo tốc độ rps mỗi giây
    go func() {
        ticker := time.NewTicker(time.Second / time.Duration(rps))
        defer ticker.Stop()
        for range ticker.C {
            select {
            case rl.tokens <- struct{}{}:
                // Thêm token vào buffer
            default:
                // Buffer đầy — bỏ qua (không block)
            }
        }
    }()

    return rl
}

// Wait blocks cho đến khi có token sẵn sàng
func (rl *RateLimiter) Wait() {
    <-rl.tokens // Block nếu không có token (rate limit hit)
}

func main() {
    limiter := NewRateLimiter(10) // 10 requests/second

    http.HandleFunc("/api", func(w http.ResponseWriter, r *http.Request) {
        limiter.Wait() // Chờ token — throttle request
        fmt.Fprint(w, "OK")
    })

    http.ListenAndServe(":8080", nil)
}

Event Bus với Pub/Sub

event_bus.go Go 
package main

import (
    "fmt"
    "sync"
)

type Event struct {
    Topic   string
    Payload any
}

// EventBus: Pub/Sub pattern với buffered channels
type EventBus struct {
    mu          sync.RWMutex
    subscribers map[string][]chan Event
}

func NewEventBus() *EventBus {
    return &EventBus{
        subscribers: make(map[string][]chan Event),
    }
}

func (eb *EventBus) Subscribe(topic string) <-chan Event {
    eb.mu.Lock()
    defer eb.mu.Unlock()

    // Buffered channel = subscriber không block publisher
    ch := make(chan Event, 100)
    eb.subscribers[topic] = append(eb.subscribers[topic], ch)
    return ch
}

func (eb *EventBus) Publish(event Event) {
    eb.mu.RLock()
    defer eb.mu.RUnlock()

    for _, ch := range eb.subscribers[event.Topic] {
        select {
        case ch <- event: // Non-blocking send
        default:
            fmt.Printf("Subscriber buffer đầy, bỏ qua event trên topic: %s\n", event.Topic)
        }
    }
}

func main() {
    bus := NewEventBus()

    orderEvents := bus.Subscribe("order.created")
    emailEvents := bus.Subscribe("order.created")

    bus.Publish(Event{Topic: "order.created", Payload: "order#123"})

    fmt.Println(<-orderEvents) // {order.created order#123}
    fmt.Println(<-emailEvents) // {order.created order#123}
}

09 — Best Practices

✅NÊN LÀM
  • Luôn có exit path cho goroutines dùng channel
  • Chỉ sender được đóng channel
  • Dùng context để cancel goroutines
  • Dùng buffered(1) khi muốn non-blocking fire-and-forget
  • Race detector: go test -race
  • Kiểm tra ok khi nhận: v, ok := <-ch
❌KHÔNG NÊN LÀM
  • Receiver không được đóng channel
  • Không gửi vào closed channel (panic!)
  • Không để goroutines leak (block mãi mãi)
  • Không dùng time.Sleep để sync
  • Không share memory qua nhiều goroutines không có mutex/channel
  • Không ignore context cancellation
🎯KHI CHỌN UNBUFFERED
  • Cần guarantee rằng sender biết receiver đã nhận
  • Signaling / synchronization (done channels)
  • Handshake protocol giữa 2 goroutines
  • Khi latency quan trọng hơn throughput
🎯KHI CHỌN BUFFERED
  • Job queues / task distribution
  • Khi sender nhanh hơn receiver
  • Semaphore / rate limiting
  • Giảm số context switches
  • Non-blocking fire-and-forget (cap=1)
🔬

Công thức chọn buffer size

Không có công thức tuyệt đối. Nguyên tắc chung: buffer size = số goroutines sender × burst factor. Profiling với go tool pprof và thử nghiệm với runtime.NumGoroutine() để tìm điểm tối ưu.

channel_cheatsheet.go Go 
// ====== CHANNEL CHEATSHEET ======

// Tạo channel
ub  := make(chan T)       // unbuffered
buf := make(chan T, N)    // buffered

// Directional channels (type safety)
var send chan<- T          // send-only
var recv <-chan T          // receive-only

// Gửi / Nhận
ch <- value               // gửi
v  := <-ch               // nhận (block)
v, ok := <-ch            // nhận + check closed

// Non-blocking operations (select + default)
select {
case ch <- v: // gửi nếu được
default:       // không block nếu đầy
}

// Drain một channel đã closed
for v := range ch { fmt.Println(v) }

// Đóng an toàn với sync.Once
var once sync.Once
once.Do(func() { close(ch) })

// Done channel pattern
done := make(chan struct{})
go func() { doWork(); close(done) }()
<-done // wait for work

// Semaphore pattern
sem := make(chan struct{}, 5) // max 5 concurrent
sem <- struct{}{}              // acquire
defer func() { <-sem }()     // release

// Timer channel
select {
case <-time.After(5 * time.Second):
    fmt.Println("timeout")
case v := <-ch:
    fmt.Println("received", v)
}
