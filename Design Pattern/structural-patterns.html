💡 Structural Patterns là gì?
Mẫu cấu trúc — Tổ chức & kết hợp object/interface hiệu quả

Structural Patterns giải quyết bài toán tổ chức các class và object thành cấu trúc lớn hơn, đảm bảo cấu trúc đó vẫn linh hoạt và hiệu quả. Không quan tâm đến cách object được tạo (Creational) hay giao tiếp (Behavioral), mà quan tâm đến cách ghép các interface và class lại.

Trong Go, structural patterns phát huy mạnh mẽ nhờ: interface duck typing (không cần khai báo explicitly), struct embedding (composition thay inheritance), và function wrapping thay vì subclassing.

Hình 3.1: Sơ đồ kiến trúc 7 Structural Design Patterns trong Go — Tối ưu hóa ghép nối, đóng gói và chia sẻ tài nguyên hệ thống

01
Adapter Pattern
Chuyển đổi interface — giúp các class không tương thích làm việc cùng nhau
STRUCTURAL
💡 Định nghĩa — GoF
Adapter — Bộ chuyển đổi interface

Adapter chuyển đổi interface của một class thành interface khác mà client mong đợi. Adapter cho phép các class với interface không tương thích làm việc cùng nhau — như ổ cắm điện 3 chân cắm vào 2 lỗ cần adapter vậy. Trong Go, pattern này cực phổ biến khi tích hợp: third-party library → domain interface, legacy API → new protocol, hoặc external service → internal abstraction.

Hai dạng: Object Adapter (dùng composition — Go preferred) và Class Adapter (dùng embedding — Go idiom).

Hình 1.1: Kiến trúc Adapter Pattern trong Go — Bộ chuyển đổi giao diện giúp kết nối linh hoạt các thành phần không tương thích

📝 Example 1 — Storage Adapter: S3 → Internal Interface (Advanced)

GOstorage_adapter.goADVANCED
// Adapter Pattern: AWS S3 SDK → internal Storage interface
// Vấn đề: AWS SDK có interface riêng, domain code dùng interface của mình
// Giải pháp: S3Adapter wrap S3 SDK, implement Storage interface

// Target interface — domain abstraction
type Storage interface {
    Upload(ctx context.Context, key string, data io.Reader) error
    Download(ctx context.Context, key string) (io.ReadCloser, error)
    Delete(ctx context.Context, key string) error
    Exists(ctx context.Context, key string) (bool, error)
}

// Adaptee: AWS S3 client (third-party, incompatible interface)
type AWSS3Client struct {
    bucket string
    client *s3.Client
}

func (a *AWSS3Client) PutObject(ctx context.Context, in *s3.PutObjectInput) (*s3.PutObjectOutput, error) {
    return a.client.PutObject(ctx, in)
}
func (a *AWSS3Client) GetObject(ctx context.Context, in *s3.GetObjectInput) (*s3.GetObjectOutput, error) {
    return a.client.GetObject(ctx, in)
}

// Adapter: S3StorageAdapter — Object Adapter pattern
// Wraps AWSS3Client, implements Storage interface
type S3StorageAdapter struct {
    adaptee *AWSS3Client
    bucket  string
    prefix  string
}

func NewS3Storage(client *s3.Client, bucket, prefix string) Storage {
    return &S3StorageAdapter{
        adaptee: &AWSS3Client{bucket: bucket, client: client},
        bucket:  bucket,
        prefix:  prefix,
    }
}

func (a *S3StorageAdapter) Upload(ctx context.Context, key string, data io.Reader) error {
    _, err := a.adaptee.PutObject(ctx, &s3.PutObjectInput{
        Bucket: aws.String(a.bucket),
        Key:    aws.String(a.prefix + key),
        Body:   data,
    })
    return wrapErr("upload", key, err)
}

func (a *S3StorageAdapter) Download(ctx context.Context, key string) (io.ReadCloser, error) {
    out, err := a.adaptee.GetObject(ctx, &s3.GetObjectInput{
        Bucket: aws.String(a.bucket),
        Key:    aws.String(a.prefix + key),
    })
    if err != nil { return nil, wrapErr("download", key, err) }
    return out.Body, nil
}

func (a *S3StorageAdapter) Delete(ctx context.Context, key string) error {
    // Translate domain call → AWS SDK call
    _, err := a.adaptee.client.DeleteObject(ctx, &s3.DeleteObjectInput{
        Bucket: aws.String(a.bucket),
        Key:    aws.String(a.prefix + key),
    })
    return wrapErr("delete", key, err)
}

func (a *S3StorageAdapter) Exists(ctx context.Context, key string) (bool, error) {
    _, err := a.adaptee.client.HeadObject(ctx, &s3.HeadObjectInput{
        Bucket: aws.String(a.bucket),
        Key:    aws.String(a.prefix + key),
    })
    if err != nil {
        var notFound *types.NoSuchKey
        if errors.As(err, ¬Found) { return false, nil }
        return false, err
    }
    return true, nil
}

func wrapErr(op, key string, err error) error {
    if err != nil { return fmt.Errorf("storage.%s(%q): %w", op, key, err) }
    return nil
}

// In-memory adapter for testing — same interface, no AWS needed
type MemStorage struct {
    mu   sync.RWMutex
    data map[string][]byte
}

func (m *MemStorage) Upload(ctx context.Context, key string, data io.Reader) error {
    b, _ := io.ReadAll(data)
    m.mu.Lock(); m.data[key] = b; m.mu.Unlock()
    return nil
}

📝 Example 2 — Two-Way Adapter: gRPC ↔ REST (Expert)

GOgrpc_rest_adapter.goEXPERT
// Two-Way Adapter: expose gRPC service as HTTP REST và ngược lại
// Một service vừa serve gRPC clients vừa serve REST clients

// Domain interface — technology-agnostic
type UserService interface {
    GetUser(ctx context.Context, id string) (*User, error)
    CreateUser(ctx context.Context, req CreateUserRequest) (*User, error)
    ListUsers(ctx context.Context, page, size int) ([]*User, int, error)
}

type User struct {
    ID, Name, Email string
    CreatedAt       time.Time
}

// gRPC Adapter: exposes UserService as gRPC server
type GRPCUserAdapter struct {
    pb.UnimplementedUserServiceServer
    svc UserService
}

func NewGRPCAdapter(svc UserService) *GRPCUserAdapter { return &GRPCUserAdapter{svc: svc} }

func (g *GRPCUserAdapter) GetUser(ctx context.Context, req *pb.GetUserRequest) (*pb.UserResponse, error) {
    user, err := g.svc.GetUser(ctx, req.Id)
    if err != nil { return nil, toGRPCError(err) }
    return toProto(user), nil
}

func (g *GRPCUserAdapter) CreateUser(ctx context.Context, req *pb.CreateUserRequest) (*pb.UserResponse, error) {
    user, err := g.svc.CreateUser(ctx, CreateUserRequest{
        Name: req.Name, Email: req.Email,
    })
    if err != nil { return nil, toGRPCError(err) }
    return toProto(user), nil
}

// HTTP REST Adapter: exposes UserService as HTTP REST
type HTTPUserAdapter struct{ svc UserService }

func NewHTTPAdapter(svc UserService) *HTTPUserAdapter { return &HTTPUserAdapter{svc: svc} }

func (h *HTTPUserAdapter) RegisterRoutes(mux *http.ServeMux) {
    mux.HandleFunc("GET /users/{id}", h.getUser)
    mux.HandleFunc("POST /users",    h.createUser)
    mux.HandleFunc("GET /users",     h.listUsers)
}

func (h *HTTPUserAdapter) getUser(w http.ResponseWriter, r *http.Request) {
    id := r.PathValue("id")
    user, err := h.svc.GetUser(r.Context(), id)
    if err != nil { writeErr(w, err); return }
    writeJSON(w, http.StatusOK, user)
}

func (h *HTTPUserAdapter) createUser(w http.ResponseWriter, r *http.Request) {
    var req CreateUserRequest
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        writeErr(w, &ErrBadRequest{Msg: err.Error()}); return
    }
    user, err := h.svc.CreateUser(r.Context(), req)
    if err != nil { writeErr(w, err); return }
    writeJSON(w, http.StatusCreated, user)
}

📝 Example 3 — Database Adapter: Multi-Driver (Expert)

GOdb_adapter.goEXPERT
// Adapter: Wrap nhiều DB drivers (pgx, sqlite, mysql) → 1 interface
// Cho phép swap DB mà không đổi application code

type Row  interface { Scan(dest ...any) error }
type Rows interface { Next() bool; Scan(...any) error; Close() error; Err() error }

type DB interface {
    QueryRow(ctx context.Context, sql string, args ...any) Row
    Query(ctx context.Context, sql string, args ...any) (Rows, error)
    Exec(ctx context.Context, sql string, args ...any) (int64, error)
    BeginTx(ctx context.Context) (Tx, error)
    Close() error
}

type Tx interface {
    Commit() error
    Rollback() error
    DB // embeds DB for queries within transaction
}

// PgxAdapter: adapts pgx.Pool → DB interface
type PgxAdapter struct{ pool *pgxpool.Pool }

func NewPgxDB(dsn string) (DB, error) {
    pool, err := pgxpool.New(context.Background(), dsn)
    if err != nil { return nil, err }
    return &PgxAdapter{pool: pool}, nil
}

func (p *PgxAdapter) QueryRow(ctx context.Context, sql string, args ...any) Row {
    return p.pool.QueryRow(ctx, sql, args...)
}

func (p *PgxAdapter) Query(ctx context.Context, sql string, args ...any) (Rows, error) {
    rows, err := p.pool.Query(ctx, sql, args...)
    return &pgxRowsAdapter{rows}, err
}

func (p *PgxAdapter) Exec(ctx context.Context, sql string, args ...any) (int64, error) {
    tag, err := p.pool.Exec(ctx, sql, args...)
    return tag.RowsAffected(), err
}

func (p *PgxAdapter) BeginTx(ctx context.Context) (Tx, error) {
    tx, err := p.pool.Begin(ctx)
    if err != nil { return nil, err }
    return &pgxTxAdapter{tx}, nil
}

func (p *PgxAdapter) Close() error { p.pool.Close(); return nil }

// pgxRowsAdapter wraps pgx.Rows → Rows interface
type pgxRowsAdapter struct{ rows pgx.Rows }
func (r *pgxRowsAdapter) Next() bool           { return r.rows.Next() }
func (r *pgxRowsAdapter) Scan(d ...any) error { return r.rows.Scan(d...) }
func (r *pgxRowsAdapter) Close() error         { r.rows.Close(); return nil }
func (r *pgxRowsAdapter) Err() error            { return r.rows.Err() }
✅

Go Best Practice: Định nghĩa interface ở phía consumer, không phải producer. Package A dùng Storage không nên import package S3 — thay vào đó A định nghĩa Storage interface và S3Adapter implement nó từ bên ngoài.

02
Decorator Pattern
Thêm chức năng vào object bằng wrapping — không cần thay đổi original class
STRUCTURAL
💡 Định nghĩa — GoF
Decorator — Wrap để mở rộng hành vi

Decorator attach additional responsibilities lên object dynamically. Decorator cung cấp alternative linh hoạt cho subclassing để mở rộng chức năng. Cả Component và Decorator implement cùng interface — client không biết đang dùng decorated hay original object. Trong Go: HTTP middleware, gRPC interceptors, io.Reader/Writer wrapping.

Khác Adapter: Adapter thay đổi interface, Decorator giữ nguyên interface nhưng thêm behavior. Khác Proxy: Decorator thêm features, Proxy kiểm soát truy cập.

Hình 2.1: Kiến trúc Decorator Pattern trong Go — Đóng gói đồng tâm giúp bổ sung hành vi (Middleware, Caching, Logging) mà không cần thay đổi mã nguồn gốc
Decorator Pattern — Stacked io.Reader Layers

📝 Example 1 — Logger + Metrics Decorator (Advanced)

GOservice_decorator.goADVANCED
// Decorator Pattern: Service với logging + metrics + retry
// Mỗi decorator wrap service, thêm 1 layer behavior

type ProductService interface {
    GetProduct(ctx context.Context, id string) (*Product, error)
    CreateProduct(ctx context.Context, p Product) (*Product, error)
    Search(ctx context.Context, q string, page, size int) ([]*Product, error)
}

// Base implementation
type productServiceImpl struct{ repo ProductRepo }

// ─── Decorator 1: Logging ──────────────────────────────────────
type loggingProductService struct {
    next   ProductService
    logger *slog.Logger
}

func WithLogging(svc ProductService, l *slog.Logger) ProductService {
    return &loggingProductService{next: svc, logger: l}
}

func (d *loggingProductService) GetProduct(ctx context.Context, id string) (*Product, error) {
    start := time.Now()
    p, err := d.next.GetProduct(ctx, id)
    d.logger.Info("GetProduct",
        "id", id, "dur", time.Since(start), "err", err)
    return p, err
}

func (d *loggingProductService) CreateProduct(ctx context.Context, p Product) (*Product, error) {
    start := time.Now()
    result, err := d.next.CreateProduct(ctx, p)
    d.logger.Info("CreateProduct",
        "name", p.Name, "dur", time.Since(start), "err", err)
    return result, err
}

func (d *loggingProductService) Search(ctx context.Context, q string, page, size int) ([]*Product, error) {
    start := time.Now()
    res, err := d.next.Search(ctx, q, page, size)
    d.logger.Info("Search",
        "q", q, "results", len(res), "dur", time.Since(start))
    return res, err
}

// ─── Decorator 2: Metrics ──────────────────────────────────────
type metricsProductService struct {
    next     ProductService
    requests *prometheus.CounterVec
    duration *prometheus.HistogramVec
}

func WithMetrics(svc ProductService, reg prometheus.Registerer) ProductService {
    d := &metricsProductService{
        next: svc,
        requests: prometheus.NewCounterVec(prometheus.CounterOpts{
            Name: "product_service_requests_total", Help: "Total requests",
        }, []string{"method", "status"}),
        duration: prometheus.NewHistogramVec(prometheus.HistogramOpts{
            Name: "product_service_duration_seconds",
        }, []string{"method"}),
    }
    reg.MustRegister(d.requests, d.duration)
    return d
}

func (d *metricsProductService) measure(method string, fn func() error) error {
    start := time.Now()
    err := fn()
    status := "ok"
    if err != nil { status = "error" }
    d.requests.WithLabelValues(method, status).Inc()
    d.duration.WithLabelValues(method).Observe(time.Since(start).Seconds())
    return err
}

func (d *metricsProductService) GetProduct(ctx context.Context, id string) (*Product, error) {
    var p *Product
    err := d.measure("GetProduct", func() error {
        var e error; p, e = d.next.GetProduct(ctx, id); return e
    })
    return p, err
}

// ─── Decorator 3: Retry ────────────────────────────────────────
func WithRetry(svc ProductService, maxAttempts int) ProductService {
    return &retryProductService{next: svc, maxAttempts: maxAttempts}
}

// ─── Wire up all decorators ────────────────────────────────────
func NewProductServiceWithAll(repo ProductRepo, logger *slog.Logger, reg prometheus.Registerer) ProductService {
    var svc ProductService = &productServiceImpl{repo: repo}
    svc = WithRetry(svc, 3)           // innermost — retry raw errors
    svc = WithMetrics(svc, reg)        // metrics include retry time
    svc = WithLogging(svc, logger)     // outermost — log everything
    return svc
}
// Client thấy ProductService — không biết gì về decorators

📝 Example 2 — io.Writer Decorator Chain (Expert)

GOio_decorator.goEXPERT
// Go stdlib thực sự dùng Decorator pattern cho io.Writer
// Custom: encrypt → compress → hash → write

// HashWriter: wrap io.Writer, tính checksum song song khi write
type HashWriter struct {
    w    io.Writer
    hash hash.Hash
}

func NewHashWriter(w io.Writer, h hash.Hash) *HashWriter { return &HashWriter{w: w, hash: h} }

func (hw *HashWriter) Write(p []byte) (int, error) {
    n, err := hw.w.Write(p)
    hw.hash.Write(p[:n]) // compute hash of written bytes
    return n, err
}

func (hw *HashWriter) Sum() []byte { return hw.hash.Sum(nil) }

// LimitWriter: wrap io.Writer, giới hạn số byte có thể write
type LimitWriter struct {
    w         io.Writer
    remaining int64
}

func NewLimitWriter(w io.Writer, limit int64) *LimitWriter {
    return &LimitWriter{w: w, remaining: limit}
}

func (lw *LimitWriter) Write(p []byte) (int, error) {
    if lw.remaining <= 0 { return 0, errors.New("write limit exceeded") }
    if int64(len(p)) > lw.remaining { p = p[:lw.remaining] }
    n, err := lw.w.Write(p)
    lw.remaining -= int64(n)
    return n, err
}

// TeeWriter: tee output to multiple writers (like io.MultiWriter but decorator)
type TeeWriter struct {
    primary   io.Writer
    secondary io.Writer
}

func (t *TeeWriter) Write(p []byte) (int, error) {
    n, err := t.primary.Write(p)
    if err != nil { return n, err }
    t.secondary.Write(p[:n]) // best-effort, ignore error
    return n, nil
}

03
Proxy Pattern
Kiểm soát truy cập — Surrogate cho object thật
STRUCTURAL
💡 Định nghĩa — GoF
Proxy — Thay thế và kiểm soát truy cập tới object

Proxy cung cấp một surrogate hoặc placeholder cho một object khác để kiểm soát truy cập tới nó. Các loại Proxy: Virtual Proxy (lazy init, tốn kém), Protection Proxy (auth/permission), Cache Proxy (cache kết quả), Remote Proxy (remote object). Proxy giữ nguyên interface với Subject thật — client không biết đang nói chuyện với proxy hay real object.

Khác Decorator: Decorator adds behavior; Proxy controls access to same behavior.

Hình 3.2: Kiến trúc Proxy Pattern trong Go — Khung bảo vệ kiểm soát quyền truy cập, xác thực và lưu bộ nhớ đệm (Cache) trước khi đi vào đối tượng xử lý thực tế
Proxy Variants — 4 loại Proxy trong Production Go

📝 Example 1 — Cache Proxy với TTL & LRU (Advanced)

GOcache_proxy.goADVANCED
// Cache Proxy: transparent caching cho repository
// Client dùng CachedProductRepo giống ProductRepo thật

type ProductRepo interface {
    FindByID(ctx context.Context, id string) (*Product, error)
    Save(ctx context.Context, p *Product) error
    Delete(ctx context.Context, id string) error
}

type cacheEntry struct {
    value     *Product
    expiresAt time.Time
}

type CachedProductRepo struct {
    real ProductRepo                  // delegate
    mu   sync.RWMutex
    data map[string]*cacheEntry
    ttl  time.Duration
    hits, misses int64               // metrics
}

func NewCachedRepo(real ProductRepo, ttl time.Duration) ProductRepo {
    r := &CachedProductRepo{real: real, data: make(map[string]*cacheEntry), ttl: ttl}
    go r.evictLoop() // background cleanup
    return r
}

func (c *CachedProductRepo) FindByID(ctx context.Context, id string) (*Product, error) {
    // Check cache first
    c.mu.RLock()
    entry, ok := c.data[id]
    c.mu.RUnlock()

    if ok && time.Now().Before(entry.expiresAt) {
        atomic.AddInt64(&c.hits, 1)
        return entry.value, nil // cache HIT
    }

    // Cache MISS — delegate to real repo
    atomic.AddInt64(&c.misses, 1)
    p, err := c.real.FindByID(ctx, id)
    if err != nil { return nil, err }

    // Populate cache
    c.mu.Lock()
    c.data[id] = &cacheEntry{value: p, expiresAt: time.Now().Add(c.ttl)}
    c.mu.Unlock()
    return p, nil
}

func (c *CachedProductRepo) Save(ctx context.Context, p *Product) error {
    if err := c.real.Save(ctx, p); err != nil { return err }
    // Invalidate cache on write
    c.mu.Lock(); delete(c.data, p.ID); c.mu.Unlock()
    return nil
}

func (c *CachedProductRepo) Delete(ctx context.Context, id string) error {
    if err := c.real.Delete(ctx, id); err != nil { return err }
    c.mu.Lock(); delete(c.data, id); c.mu.Unlock()
    return nil
}

func (c *CachedProductRepo) Stats() (int64, int64) {
    return atomic.LoadInt64(&c.hits), atomic.LoadInt64(&c.misses)
}

func (c *CachedProductRepo) evictLoop() {
    t := time.NewTicker(c.ttl / 2)
    for range t.C {
        now := time.Now()
        c.mu.Lock()
        for k, v := range c.data {
            if now.After(v.expiresAt) { delete(c.data, k) }
        }
        c.mu.Unlock()
    }
}

📝 Example 2 — Protection Proxy + Virtual Proxy (Expert)

GOprotection_proxy.goEXPERT
// Protection Proxy: auth + RBAC cho service method
// Virtual Proxy: lazy init cho heavy database connection

// ─── Protection Proxy ─────────────────────────────────────────
type Permission string
const (PermRead Permission = "read"; PermWrite Permission = "write"; PermAdmin Permission = "admin")

type AuthContext struct {
    UserID      string
    Permissions []Permission
}

func (a *AuthContext) Has(p Permission) bool {
    for _, perm := range a.Permissions { if perm == p { return true } }
    return false
}

type authProductRepo struct {
    real    ProductRepo
    authCtx func(context.Context) (*AuthContext, error)
}

func WithAuth(real ProductRepo, getAuth func(context.Context) (*AuthContext, error)) ProductRepo {
    return &authProductRepo{real: real, authCtx: getAuth}
}

func (a *authProductRepo) FindByID(ctx context.Context, id string) (*Product, error) {
    auth, err := a.authCtx(ctx)
    if err != nil { return nil, fmt.Errorf("auth: %w", err) }
    if !auth.Has(PermRead) { return nil, ErrForbidden }
    return a.real.FindByID(ctx, id)
}

func (a *authProductRepo) Save(ctx context.Context, p *Product) error {
    auth, err := a.authCtx(ctx)
    if err != nil { return fmt.Errorf("auth: %w", err) }
    if !auth.Has(PermWrite) { return ErrForbidden }
    return a.real.Save(ctx, p)

    if err != nil { return err }
    if !auth.Has(PermAdmin) { return ErrForbidden }
    return a.real.Delete(ctx, id)
}

Facade Pattern
Cổng vào đơn giản cho hệ thống phức tạp — giảm coupling
STRUCTURAL
💡 Định nghĩa — GoF
Facade — Giao diện thống nhất cho một tập subsystem

Facade cung cấp một high-level interface đơn giản cho một tập hợp interface phức tạp trong một subsystem. Facade định nghĩa higher-level interface làm cho subsystem dễ sử dụng hơn. Client không cần biết tất cả các class bên trong — chỉ cần biết Facade.

Trong Go: package-level API functions (như http.ListenAndServe là Facade cho toàn bộ net/http), log.Fatal, service layer che giấu repository/cache/queue phức tạp.

Hình 4.1: Kiến trúc Facade Pattern trong Go — Cung cấp một cổng giao diện thống nhất, tinh gọn làm giảm mức độ liên kết (Coupling) giữa Client và nhiều hệ thống con phức tạp bên dưới

// Che giấu phức tạp của 5 subsystem sau 1 API đơn giản

    TrackingNo string

// Facade — single entry point
type OrderFacade struct {
    inventory    InventoryService
    payment      PaymentService
    shipping     ShippingService
    notification NotificationService
    audit        AuditService
    orders       OrderRepository
}

func NewOrderFacade(
    inv InventoryService, pay PaymentService,
    ship ShippingService, notif NotificationService,
    audit AuditService, repo OrderRepository,
) *OrderFacade {
    return &OrderFacade{
        inventory: inv, payment: pay, shipping: ship,
        notification: notif, audit: audit, orders: repo,
    }
}

// PlaceOrder: orchestrate toàn bộ flow phức tạp sau 1 method call
func (f *OrderFacade) PlaceOrder(ctx context.Context, req PlaceOrderRequest) (*PlaceOrderResult, error) {
    // Step 1: Reserve inventory (saga compensation)
    reservationID, err := f.inventory.Reserve(ctx, req.Items)
    if err != nil { return nil, fmt.Errorf("inventory: %w", err) }

    // Step 2: Charge payment
    paymentID, err := f.payment.Charge(ctx, req.PayToken, f.totalAmount(req.Items))
    if err != nil {
        f.inventory.Release(ctx, reservationID) // rollback
        return nil, fmt.Errorf("payment: %w", err)
    }

    // Step 3: Create order record
    order, err := f.orders.Create(ctx, Order{
        CustomerID:    req.CustomerID,
        Items:         req.Items,
        PaymentID:     paymentID,
        ReservationID: reservationID,
    })
    if err != nil {
        f.payment.Refund(ctx, paymentID)
        f.inventory.Release(ctx, reservationID)
        return nil, fmt.Errorf("create order: %w", err)
    }

    // Step 4: Create shipment
    shipment, err := f.shipping.CreateShipment(ctx, ShipmentRequest{
        OrderID: order.ID, Items: req.Items, Addr: req.ShipAddr,
    })
    if err != nil {
        log.Printf("shipping failed for order %s: %v", order.ID, err)
        // Non-fatal: order exists, ship later
    }

    // Step 5: Notify customer (async, non-blocking)
    go f.notification.SendOrderConfirmation(context.Background(), NotifReq{
        CustomerID: req.CustomerID, OrderID: order.ID,
    })

    // Step 6: Audit log
    f.audit.Log(ctx, AuditEvent{
        Action: "order.placed", ActorID: req.CustomerID, ResourceID: order.ID,
    })

    return &PlaceOrderResult{
        OrderID:     order.ID,
        TrackingNo:  shipment.TrackingNumber(),
        EstDelivery: shipment.EstimatedDelivery(),
    }, nil
}

func (f *OrderFacade) CancelOrder(ctx context.Context, orderID string) error {
    order, err := f.orders.FindByID(ctx, orderID)
    if err != nil { return err }

    var errs []error
    errs = append(errs, f.payment.Refund(ctx, order.PaymentID))
    errs = append(errs, f.inventory.Release(ctx, order.ReservationID))
    errs = append(errs, f.shipping.Cancel(ctx, orderID))
    errs = append(errs, f.orders.UpdateStatus(ctx, orderID, "cancelled"))

    go f.notification.SendCancellation(context.Background(), order.CustomerID, orderID)

    return errors.Join(errs...)
}

func (f *OrderFacade) totalAmount(items []OrderItem) int64 {
    var total int64
    for _, item := range items { total += item.Price * int64(item.Qty) }
    return total
}

Composite Pattern
Cấu trúc cây — xử lý leaf và composite như nhau
STRUCTURAL
💡 Định nghĩa — GoF
Composite — Part-whole hierarchy với giao diện đồng nhất

Composite tổ chức objects thành cấu trúc cây để represent part-whole hierarchies. Composite cho phép client xử lý individual objects (leaf) và compositions (composite) theo cùng một cách qua interface chung. Ví dụ điển hình: file system (file & folder), UI components (widget & container), menu items, arithmetic expression trees.

Hình 5.1: Kiến trúc Composite Pattern trong Go — Biểu diễn cấu trúc cây phân cấp (Part-Whole) cho phép tương tác đồng nhất giữa nút lá (Leaf) và nút phức hợp (Composite Node)

// Composite Pattern: File System với unified interface
// Client gọi Size() / Print() trên cả File và Directory giống nhau

type FSComponent interface {
    Name() string
    Size() int64
    Print(indent string)
    Find(name string) FSComponent
}

// Leaf: File
type File struct {
    name    string
    size    int64
    content []byte
}

func NewFile(name string, content []byte) *File {
    return &File{name: name, size: int64(len(content)), content: content}
}

func (f *File) Name() string               { return f.name }
func (f *File) Size() int64                { return f.size }
func (f *File) Find(name string) FSComponent {
    if f.name == name { return f }; return nil
}
func (f *File) Print(indent string) {
    fmt.Printf("%s📄 %s (%d bytes)\n", indent, f.name, f.size)
}

// Composite: Directory
type Directory struct {
    name     string
    children []FSComponent
}

func NewDirectory(name string) *Directory { return &Directory{name: name} }

func (d *Directory) Add(c FSComponent) *Directory {
    d.children = append(d.children, c); return d
}

func (d *Directory) Remove(name string) {
    for i, c := range d.children {
        if c.Name() == name {
            d.children = append(d.children[:i], d.children[i+1:]...)
            return
        }
    }
}

func (d *Directory) Name() string { return d.name }

func (d *Directory) Size() int64 {
    var total int64
    for _, c := range d.children { total += c.Size() } // recursive
    return total
}

func (d *Directory) Print(indent string) {
    fmt.Printf("%s📁 %s/ (%d bytes)\n", indent, d.name, d.Size())
    for _, c := range d.children {
        c.Print(indent + "  ") // recursive — both File and Dir handled same way
    }
}

func (d *Directory) Find(name string) FSComponent {
    if d.name == name { return d }
    for _, c := range d.children {
        if found := c.Find(name); found != nil { return found }
    }
    return nil
}

// Usage
func main() {
    root := NewDirectory("root")
    src  := NewDirectory("src")
    src.Add(NewFile("main.go",   []byte("package main...")))
    src.Add(NewFile("config.go", []byte("package main...")))
    root.Add(src).Add(NewFile("go.mod", []byte("module app")))
    root.Print("")
    fmt.Printf("Total: %d bytes\n", root.Size())
}

// Composite Pattern: Rule Engine
// Leaf = simple rule, Composite = AND/OR group of rules
// Client evaluate() tree without knowing if it's leaf or group

type RuleContext struct {
    User    User
    Product Product
    Cart    Cart
    Meta    map[string]any
}

type Rule interface {
    Evaluate(ctx RuleContext) (bool, string) // (match, reason)
    Name() string
}

// Leaf rules
type AgeRule struct{ MinAge int }
func (r AgeRule) Name() string { return fmt.Sprintf("age >= %d", r.MinAge) }
func (r AgeRule) Evaluate(ctx RuleContext) (bool, string) {
    ok := ctx.User.Age >= r.MinAge
    if !ok { return false, fmt.Sprintf("user age %d < %d", ctx.User.Age, r.MinAge) }
    return true, ""
}

type MinCartRule struct{ Min float64 }
func (r MinCartRule) Name() string { return fmt.Sprintf("cart >= $%.2f", r.Min) }
func (r MinCartRule) Evaluate(ctx RuleContext) (bool, string) {
    ok := ctx.Cart.Total >= r.Min
    if !ok { return false, fmt.Sprintf("cart $%.2f < $%.2f", ctx.Cart.Total, r.Min) }
    return true, ""
}

// Composite: AndRule — all children must pass
type AndRule struct {
    name     string
    children []Rule
}

func And(name string, rules ...Rule) *AndRule { return &AndRule{name: name, children: rules} }
func (r *AndRule) Name() string { return r.name }
func (r *AndRule) Evaluate(ctx RuleContext) (bool, string) {
    for _, child := range r.children {
        ok, reason := child.Evaluate(ctx)
        if !ok { return false, fmt.Sprintf("[%s] %s", child.Name(), reason) }
    }
    return true, ""
}

// Composite: OrRule — at least one child must pass
type OrRule struct{ name string; children []Rule }
func Or(name string, rules ...Rule) *OrRule { return &OrRule{name: name, children: rules} }
func (r *OrRule) Name() string { return r.name }
func (r *OrRule) Evaluate(ctx RuleContext) (bool, string) {
    reasons := []string{}
    for _, child := range r.children {
        ok, reason := child.Evaluate(ctx)
        if ok { return true, "" }
        reasons = append(reasons, reason)
    }
    return false, strings.Join(reasons, " AND ")
}

Bridge Pattern
Tách Abstraction khỏi Implementation — hai chiều thay đổi độc lập
STRUCTURAL
💡 Định nghĩa — GoF
Bridge — Cầu nối để tránh class hierarchy bùng nổ

Bridge tách rời một abstraction khỏi implementation để cả hai có thể thay đổi độc lập. Thay vì kế thừa (một class kết hợp cả abstraction và implementation), Bridge dùng composition: abstraction giữ reference đến implementation object. Điều này tránh "cartesian product" class explosion khi có N abstraction × M implementation.

Ví dụ kinh điển: 3 Platform (Windows, macOS, Linux) × 3 Renderer (OpenGL, DirectX, Vulkan) → thay vì 9 subclass, Bridge chỉ cần 3+3 = 6 class.

Hình 6.1: Kiến trúc Bridge Pattern trong Go — Phân tách tường minh phần trừu tượng (Abstraction) khỏi phần triển khai (Implementation) thông qua liên kết Composition thay vì Inheritance

Flyweight Pattern
Chia sẻ trạng thái chung — tiết kiệm bộ nhớ khi tạo hàng triệu object
STRUCTURAL
💡 Định nghĩa — GoF
Flyweight — Chia sẻ intrinsic state để giảm memory footprint

Flyweight dùng sharing để hỗ trợ hiệu quả một số lượng lớn fine-grained objects. Ý tưởng cốt lõi: tách Intrinsic State (shared, immutable — màu chữ, font, texture) khỏi Extrinsic State (unique per instance — vị trí, size). Flyweight Factory quản lý pool các flyweight objects và tái sử dụng chúng. Trong Go, pattern này hay thấy trong: string interning, connection pool, icon/image cache, particle system.

Khi nào dùng: Ứng dụng dùng số lượng rất lớn object (hàng nghìn/triệu), nhiều object có state giống nhau, extrinsic state có thể tách ra được.

Hình 7.1: Phân tách rõ ràng trạng thái nội tại (Intrinsic State) có thể chia sẻ và trạng thái bên ngoài (Extrinsic State) để tối ưu hóa tài nguyên bộ nhớ
Flyweight Pattern — Intrinsic vs Extrinsic State

📝 Example 1 — Text Editor Character Flyweight (Basic)

GOtext_flyweight.goBASIC
// Flyweight: Text editor characters
// Intrinsic (shared): font, size, color — identical across many chars
// Extrinsic (per instance): position (row, col) — unique per character

// Flyweight: shared character format
type CharFormat struct {
    Font  string
    Size  int
    Color string
    Bold  bool
}

// Flyweight Factory: ensures shared instances
type CharFormatFactory struct {
    mu      sync.RWMutex
    formats map[string]*CharFormat
}

var DefaultFactory = &CharFormatFactory{formats: make(map[string]*CharFormat)}

func (f *CharFormatFactory) Get(font string, size int, color string, bold bool) *CharFormat {
    key := fmt.Sprintf("%s-%d-%s-%v", font, size, color, bold)
    f.mu.RLock()
    if fw, ok := f.formats[key]; ok {
        f.mu.RUnlock()
        return fw // reuse existing flyweight
    }
    f.mu.RUnlock()
    f.mu.Lock(); defer f.mu.Unlock()
    // double-check after acquiring write lock
    if fw, ok := f.formats[key]; ok { return fw }
    fw := &CharFormat{Font: font, Size: size, Color: color, Bold: bold}
    f.formats[key] = fw
    return fw
}

func (f *CharFormatFactory) Count() int {
    f.mu.RLock(); defer f.mu.RUnlock()
    return len(f.formats)
}

// Context: Character — holds extrinsic state + ref to shared flyweight
type Character struct {
    Glyph  rune         // the actual char
    Row    int          // extrinsic: position
    Col    int          // extrinsic: position
    Format *CharFormat  // intrinsic: shared flyweight (pointer!)
}

func (c *Character) Render() string {
    bold := ""; if c.Format.Bold { bold = "**" }
    return fmt.Sprintf("[%s%c%s @(%d,%d) %s/%d/%s]",
        bold, c.Glyph, bold, c.Row, c.Col, c.Format.Font, c.Format.Size, c.Format.Color)
}

// TextDocument: collection of characters
type TextDocument struct {
    chars   []*Character
    factory *CharFormatFactory
}

func NewTextDocument() *TextDocument {
    return &TextDocument{factory: DefaultFactory}
}

func (d *TextDocument) AddChar(ch rune, row, col int, font string, size int, color string, bold bool) {
    fmt := d.factory.Get(font, size, color, bold) // reuse shared format
    d.chars = append(d.chars, &Character{Glyph: ch, Row: row, Col: col, Format: fmt})
}

func (d *TextDocument) Stats() {
    fmt.Printf("Chars: %d  |  Unique formats: %d\n", len(d.chars), d.factory.Count())
    // 10000 chars, same format → 10000 chars sharing 1 CharFormat object
}

// Usage:
// doc := NewTextDocument()
// for i, ch := range "Hello World" × 1000 {
//     doc.AddChar(ch, i/80, i%80, "Arial", 12, "black", false)
// }

📝 Example 2 — Connection Pool Flyweight (Advanced)

GOconn_pool_flyweight.goADVANCED
// Flyweight Pattern: Database connection pooling
// Intrinsic: connection params (DSN, driver, TLS config) — shared
// Extrinsic: current transaction, timeout per request — caller-managed

type DBParams struct {
    Driver   string
    DSN      string
    MaxConns int
    TLSMode  string
}

// Flyweight: shared DB pool (intrinsic config)
type DBPool struct {
    params   DBParams
    pool     *sql.DB
    createdAt time.Time
}

func (p *DBPool) Key() string {
    return fmt.Sprintf("%s://%s", p.params.Driver, p.params.DSN)
}

// Flyweight Factory: connection pool registry
type PoolRegistry struct {
    mu    sync.RWMutex
    pools map[string]*DBPool
}

var GlobalRegistry = &PoolRegistry{pools: make(map[string]*DBPool)}

func (r *PoolRegistry) Get(params DBParams) (*DBPool, error) {
    key := fmt.Sprintf("%s://%s", params.Driver, params.DSN)

    r.mu.RLock()
    if p, ok := r.pools[key]; ok {
        r.mu.RUnlock()
        return p, nil // reuse shared pool
    }
    r.mu.RUnlock()

    r.mu.Lock(); defer r.mu.Unlock()
    if p, ok := r.pools[key]; ok { return p, nil }

    db, err := sql.Open(params.Driver, params.DSN)
    if err != nil { return nil, fmt.Errorf("open db: %w", err) }
    db.SetMaxOpenConns(params.MaxConns)
    db.SetMaxIdleConns(params.MaxConns / 2)
    db.SetConnMaxLifetime(30 * time.Minute)

    pool := &DBPool{params: params, pool: db, createdAt: time.Now()}
    r.pools[key] = pool
    log.Printf("created new pool for %s (total pools: %d)", key, len(r.pools))
    return pool, nil
}

func (r *PoolRegistry) Stats() map[string]sql.DBStats {
    r.mu.RLock(); defer r.mu.RUnlock()
    stats := make(map[string]sql.DBStats)
    for k, p := range r.pools { stats[k] = p.pool.Stats() }
    return stats
}

func (r *PoolRegistry) Close() {
    r.mu.Lock(); defer r.mu.Unlock()
    for _, p := range r.pools { p.pool.Close() }
}

// Context: Repository uses pool as flyweight
// Extrinsic: ctx, query timeout — passed at call time
type Repository struct {
    pool *DBPool
}

func NewRepository(params DBParams) (*Repository, error) {
    pool, err := GlobalRegistry.Get(params)
    return &Repository{pool: pool}, err
}

func (r *Repository) QueryRow(ctx context.Context, sql string, args ...any) *sql.Row {
    // extrinsic ctx carries per-request timeout/cancellation
    return r.pool.pool.QueryRowContext(ctx, sql, args...)
}
// 100 Repositories sharing the same DBPool → O(1) connections, O(n) repos

📝 Example 3 — Particle System Flyweight (Expert)

GOparticle_flyweight.goEXPERT
// Flyweight Pattern: Game particle system
// Hàng triệu particles, chỉ vài kiểu — chia sẻ texture/color/physics
// extrinsic: position, velocity, lifetime — per-particle (nhẹ, chỉ ~24 bytes)

// Flyweight: shared particle type data (heavy — texture, shader, etc.)
type ParticleType struct {
    Name       string
    Color      [4]uint8    // RGBA
    Texture    []byte      // shared texture data (can be MBs)
    Mass       float32
    Drag       float32
    Lifespan   float32    // default seconds
}

// Flyweight Factory
type ParticleTypeRegistry struct {
    types map[string]*ParticleType
}

func NewRegistry() *ParticleTypeRegistry {
    r := &ParticleTypeRegistry{types: make(map[string]*ParticleType)}
    r.Register(&ParticleType{Name: "fire",   Color: [4]uint8{255,80,0,200},   Mass: 0.01, Drag: 0.98, LifeSpan: 1.5})
    r.Register(&ParticleType{Name: "smoke",  Color: [4]uint8{128,128,128,100}, Mass: 0.001, Drag: 0.95, LifeSpan: 3.0})
    r.Register(&ParticleType{Name: "spark",  Color: [4]uint8{255,255,0,255},   Mass: 0.05, Drag: 0.99, LifeSpan: 0.5})
    r.Register(&ParticleType{Name: "water",  Color: [4]uint8{0,100,255,180},  Mass: 1.0,  Drag: 0.99, LifeSpan: 2.0})
    return r
}

func (r *ParticleTypeRegistry) Register(t *ParticleType) { r.types[t.Name] = t }
func (r *ParticleTypeRegistry) Get(name string) *ParticleType { return r.types[name] }

// Particle: extremely lightweight context — only extrinsic state
type Particle struct {
    X, Y, Z    float32  // position (extrinsic)
    VX, VY, VZ float32  // velocity (extrinsic)
    Life       float32  // remaining life (extrinsic)
    ptype      *ParticleType // flyweight reference (just 8 bytes ptr)
}

// Particle size: 7×4 + 8 = 36 bytes
// Without flyweight: 36 + texture(~1MB) per particle → 1M particles = 1TB!

func (p *Particle) Update(dt float32) bool {
    // Use intrinsic physics from flyweight
    p.VX *= p.ptype.Drag; p.VY *= p.ptype.Drag; p.VZ *= p.ptype.Drag
    p.VY -= 9.8 * p.ptype.Mass * dt // gravity
    p.X += p.VX * dt; p.Y += p.VY * dt; p.Z += p.VZ * dt
    p.Life -= dt
    return p.Life > 0
}

func (p *Particle) Color() [4]uint8 {
    c := p.ptype.Color
    c[3] = uint8(float32(c[3]) * (p.Life / p.ptype.LifeSpan)) // fade out
    return c
}

// ParticleSystem: manages millions of lightweight particles
type ParticleSystem struct {
    registry  *ParticleTypeRegistry
    particles []Particle // value slice — no alloc per particle
    pool      []Particle // pre-allocated pool
}

func NewParticleSystem(reg *ParticleTypeRegistry, capacity int) *ParticleSystem {
    return &ParticleSystem{
        registry:  reg,
        particles: make([]Particle, 0, capacity),
    }
}

func (ps *ParticleSystem) Emit(typeName string, x, y, z, vx, vy, vz float32) {
    pt := ps.registry.Get(typeName)
    if pt == nil { return }
    ps.particles = append(ps.particles, Particle{
        X: x, Y: y, Z: z, VX: vx, VY: vy, VZ: vz,
        Life: pt.LifeSpan, ptype: pt, // share flyweight
    })
}

func (ps *ParticleSystem) Update(dt float32) {
    alive := ps.particles[:0] // reuse backing array, avoid alloc
    for i := range ps.particles {
        if ps.particles[i].Update(dt) {
            alive = append(alive, ps.particles[i])
        }
    }
    ps.particles = alive
}

// Emit 1M particles: 36 bytes each = ~36MB (shared 4 ParticleTypes = ~4MB textures)
// Without flyweight: 1M × (36 + 1MB texture) ≈ 1TB — impossible
⚠️

Flyweight Pitfall: Flyweight object phải immutable (thread-safe) vì được share bởi nhiều goroutine. Không bao giờ để Flyweight có mutable state. Nếu cần modify, tạo new flyweight thay vì sửa existing. Flyweight Factory cần concurrent-safe (dùng sync.RWMutex hoặc sync.Map).

⚡ Pattern Combination Examples

🔌 + 🎨 Adapter + Decorator

Adapt third-party service → domain interface, then decorate với logging/metrics. Thứ tự: WithLogging(NewS3Adapter(s3Client))

🛡️ + 🎨 Proxy + Decorator

Cache Proxy (result memoization) bên trong, Logging Decorator bên ngoài: WithLog(NewCacheProxy(real))

🏛️ + 🌲 Facade + Composite

Facade expose simple API, internally uses Composite tree để process rules/validation step.

🌉 + 🎨 Bridge + Decorator

Bridge tách platform/renderer, Decorator thêm logging hoặc profiling lên implementation side.

Anti-Patterns cần tránh

❌ God Facade

Facade trở thành God Object khi expose quá nhiều hoặc có business logic bên trong. Facade nên chỉ orchestrate, không chứa logic.

❌ Decorator Hell

Stack quá nhiều decorator layers làm stack trace khó đọc và debug khó. Mỗi layer cần có 1 trách nhiệm rõ ràng. Không nên có hơn 5-6 layers.

❌ Proxy Leaking

Proxy expose type của real object (via type assertion). Client không bao giờ nên biết hoặc cast về real type — đây là vi phạm encapsulation.

❌ Mutable Flyweight

Flyweight bị sửa bởi một client ảnh hưởng tất cả clients. Flyweight phải immutable sau khi tạo. Nếu cần modify, tạo new flyweight instance qua factory.

✅

Go-specific summary: Trong Go, composition + interface là nền tảng tự nhiên cho structural patterns. Không cần class hierarchy phức tạp. Quy tắc vàng: "Accept interfaces, return concrete types" (Rob Pike). Định nghĩa interface ở phía consumer. Dùng struct embedding thay inheritance. Flyweight qua sync.Map hoặc sync.RWMutex + map cho thread safety.

Go-specific insight: Trong Go, Adapter, Decorator, và Proxy trông rất giống nhau về cấu trúc code (đều wrap một struct và implement cùng interface). Cách phân biệt: Adapter = interface không khớp (phải convert), Decorator = thêm behavior vào cùng interface, Proxy = kiểm soát access vào cùng behavior. Mục đích quyết định pattern, không phải cấu trúc code.
