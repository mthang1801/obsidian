Dynamic Programming là gì?

📌 Định nghĩa

Dynamic Programming (DP) là một kỹ thuật tối ưu hóa thuật toán dựa trên việc phân rã bài toán lớn thành các bài toán con nhỏ hơn, lưu trữ kết quả của chúng để tránh tính toán lại. DP áp dụng khi bài toán có hai tính chất: Optimal Substructure (cấu trúc con tối ưu) và Overlapping Subproblems (bài toán con lặp lại).

🏗️ Optimal Substructure

Giải pháp tối ưu của bài toán chứa giải pháp tối ưu của các bài toán con. Ví dụ: Đường đi ngắn nhất từ A→C = Đường đi ngắn nhất A→B + B→C.

🔁 Overlapping Subproblems

Cùng một bài toán con được giải quyết nhiều lần. DP lưu kết quả (cache) để tái sử dụng thay vì tính lại → giảm độ phức tạp từ exponential xuống polynomial.

DP Core Concepts

Hai phương pháp tiếp cận

🔽 Top-Down (Memoization)

Đệ quy + lưu kết quả vào cache (map hoặc array). Dễ implement, chỉ tính các bài toán con cần thiết. Phù hợp khi không gian trạng thái thưa.

GO memoization.go Basic 
// Top-Down: Đệ quy + Memo cache
func fibMemo(n int, memo map[int]int) int {
    if n <= 1 { return n }
    if v, ok := memo[n]; ok {
        return v // Tái sử dụng kết quả
    }
    memo[n] = fibMemo(n-1, memo) +
               fibMemo(n-2, memo)
    return memo[n]
}

🔼 Bottom-Up (Tabulation)

Lặp từ bài toán nhỏ nhất lên bài toán lớn, điền vào bảng DP. Hiệu năng tốt hơn (không có overhead đệ quy), phù hợp khi cần giải tất cả bài toán con.

GO tabulation.go Basic 
// Bottom-Up: Điền bảng từ dưới lên
func fibTable(n int) int {
    if n <= 1 { return n }
    dp := make([]int, n+1)
    dp[0], dp[1] = 0, 1
    for i := 2; i <= n; i++ {
        dp[i] = dp[i-1] + dp[i-2]
        // Điền bảng tuần tự
    }
    return dp[n]
}
💡 Khi nào dùng cái nào? Top-Down khi không gian trạng thái thưa và không phải tính hết (lazy evaluation). Bottom-Up khi cần hiệu năng tối đa và tính tất cả trạng thái.

Generic DP Framework trong Go

GO dp_framework.go Expert 
package dp

// Generic Memoizer với Go Generics
type Memoizer[K comparable, V any] struct {
    cache  map[K]V
    solver func(K) V
}

func NewMemoizer[K comparable, V any](
    solver func(K) V,
) *Memoizer[K, V] {
    m := &Memoizer[K, V]{
        cache:  make(map[K]V),
        solver: solver,
    }
    return m
}

func (m *Memoizer[K, V]) Solve(key K) V {
    if v, ok := m.cache[key]; ok {
        return v
    }
    result := m.solver(key)
    m.cache[key] = result
    return result
}

// Sử dụng với Fibonacci
func ExampleGenericMemo() {
    var memo *Memoizer[int, int]
    memo = NewMemoizer(func(n int) int {
        if n <= 1 { return n }
        return memo.Solve(n-1) + memo.Solve(n-2)
    })
    _ = memo.Solve(50) // fib(50) nhanh chóng
}

Space Optimization

📐 Tối ưu không gian

Nhiều bài toán DP chỉ cần trạng thái của layer trước, không cần toàn bộ bảng. Kỹ thuật rolling array giảm O(n²) space → O(n) hoặc O(1).

DP with Bitmask

🎭 Bitmask DP

Dùng bitmask để biểu diễn tập hợp trạng thái. State: dp[mask][i] = optimal khi đã thăm các node trong mask và đang ở node i. Dùng cho bài toán TSP, assignment problem.

DP với Stack Monotone

📊 Monotone Stack/Queue Optimization

Kết hợp DP với monotone queue để giảm độ phức tạp từ O(n²) → O(n) cho các bài toán sliding window maximum/minimum.

GO fibonacci_basic.go Basic — Memoization 
package fibonacci

import "sync"

// === APPROACH 1: Top-Down Memoization (Thread-safe) ===
type FibMemo struct {
    mu    sync.RWMutex
    cache map[int]uint64
}

func NewFibMemo() *FibMemo {
    return &FibMemo{
        cache: map[int]uint64{0: 0, 1: 1},
    }
}

func (f *FibMemo) Fib(n int) uint64 {
    f.mu.RLock()
    if v, ok := f.cache[n]; ok {
        f.mu.RUnlock()
        return v
    }
    f.mu.RUnlock()

    result := f.Fib(n-1) + f.Fib(n-2)

    f.mu.Lock()
    f.cache[n] = result
    f.mu.Unlock()
    return result
}

// === APPROACH 2: Bottom-Up Tabulation ===
func FibTable(n int) uint64 {
    if n <= 1 { return uint64(n) }
    dp := make([]uint64, n+1)
    dp[0], dp[1] = 0, 1
    for i := 2; i <= n; i++ {
        dp[i] = dp[i-1] + dp[i-2]
    }
    return dp[n]
}

// === APPROACH 3: Space Optimized O(1) ===
func FibOptimized(n int) uint64 {
    if n <= 1 { return uint64(n) }
    a, b := uint64(0), uint64(1)
    for i := 2; i <= n; i++ {
        a, b = b, a+b
    }
    return b
}
GO fibonacci_matrix.go Expert — Matrix Exponentiation O(log n) 
package fibonacci

// Matrix Exponentiation: F(n) trong O(log n) time
// Dựa trên: [[1,1],[1,0]]^n = [[F(n+1), F(n)],[F(n), F(n-1)]]

type Matrix2x2 [2][2]uint64

func matMul(a, b Matrix2x2) Matrix2x2 {
    return Matrix2x2{
        {a[0][0]*b[0][0] + a[0][1]*b[1][0],
         a[0][0]*b[0][1] + a[0][1]*b[1][1]},
        {a[1][0]*b[0][0] + a[1][1]*b[1][0],
         a[1][0]*b[0][1] + a[1][1]*b[1][1]},
    }
}

func matPow(m Matrix2x2, n int) Matrix2x2 {
    result := Matrix2x2{{1, 0}, {0, 1}} // Identity
    for n > 0 {
        if n%2 == 1 {
            result = matMul(result, m)
        }
        m = matMul(m, m) // Square the matrix
        n /= 2
    }
    return result
}

// FibMatrix: O(log n) time, O(1) space — Expert level
func FibMatrix(n int) uint64 {
    if n <= 1 { return uint64(n) }
    base := Matrix2x2{{1, 1}, {1, 0}}
    result := matPow(base, n-1)
    return result[0][0]
}

// Benchmark comparison test
func BenchmarkFib(n int) (uint64, uint64, uint64) {
    return FibTable(n), FibOptimized(n), FibMatrix(n)
}
🔑 Key Insight Matrix Exponentiation giúp tính F(10^18) trong O(log n) = ~60 phép nhân ma trận, trong khi Bottom-Up cần 10^18 bước. Đây là kỹ thuật expert dùng trong competitive programming.
02
Longest Common Subsequence
⭐⭐ Advanced

📌 Định nghĩa

LCS tìm subsequence dài nhất xuất hiện trong cả hai chuỗi. Subsequence khác Substring: các ký tự không cần liên tiếp nhưng phải giữ thứ tự tương đối. Ứng dụng: diff tools (git diff), bioinformatics (DNA alignment), spell checker.

dp[i][j] = dp[i-1][j-1] + 1   nếu s1[i] == s2[j]
dp[i][j] = max(dp[i-1][j], dp[i][j-1])   nếu s1[i] ≠ s2[j]

GO lcs.go Advanced — Full LCS với Backtracking 
package lcs

// LCSResult chứa cả độ dài và chuỗi LCS thực tế
type LCSResult struct {
    Length int
    Seq    string
}

// LCS Bottom-Up với backtracking để tìm chuỗi thực
func LCS(s1, s2 string) LCSResult {
    m, n := len(s1), len(s2)
    // Tạo bảng DP (m+1) x (n+1)
    dp := make([][]int, m+1)
    for i := range dp {
        dp[i] = make([]int, n+1)
    }

    // Điền bảng
    for i := 1; i <= m; i++ {
        for j := 1; j <= n; j++ {
            if s1[i-1] == s2[j-1] {
                dp[i][j] = dp[i-1][j-1] + 1
            } else {
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
            }
        }
    }

    // Backtrack để tìm chuỗi LCS thực tế
    seq := backtrack(dp, s1, s2, m, n)
    return LCSResult{Length: dp[m][n], Seq: seq}
}

func backtrack(dp [][]int, s1, s2 string, i, j int) string {
    if i == 0 || j == 0 { return "" }
    if s1[i-1] == s2[j-1] {
        return backtrack(dp, s1, s2, i-1, j-1) +
               string(s1[i-1])
    }
    if dp[i-1][j] > dp[i][j-1] {
        return backtrack(dp, s1, s2, i-1, j)
    }
    return backtrack(dp, s1, s2, i, j-1)
}

// Space Optimized: O(min(m,n)) space
func LCSSpaceOpt(s1, s2 string) int {
    if len(s1) < len(s2) {
        s1, s2 = s2, s1 // s2 là chuỗi ngắn hơn
    }
    m, n := len(s1), len(s2)
    prev := make([]int, n+1)
    curr := make([]int, n+1)
    for i := 1; i <= m; i++ {
        for j := 1; j <= n; j++ {
            if s1[i-1] == s2[j-1] {
                curr[j] = prev[j-1] + 1
            } else {
                curr[j] = max(prev[j], curr[j-1])
            }
        }
        prev, curr = curr, prev
    }
    return prev[n]
}
GO lcs_expert.go Expert — Hunt-Szymanski Algorithm (Sparse LCS) 
package lcs

import "sort"

// Hunt-Szymanski: Hiệu quả khi LCS thưa (sparse matches)
// Dùng trong git diff, biological sequence alignment
// Time: O((r+n) log n) trong đó r = số điểm match

func HuntSzymanski(s1, s2 string) int {
    // Bước 1: Build match positions map
    pos := make(map[byte][]int)
    for j, c := range []byte(s2) {
        pos[c] = append(pos[c], j+1)
    }

    // Bước 2: Collect all match pairs (i,j)
    type pair struct{ i, j int }
    var matches []pair
    for i, c := range []byte(s1) {
        for _, j := range pos[c] {
            matches = append(matches, pair{i + 1, j})
        }
    }
    sort.Slice(matches, func(a, b int) bool {
        if matches[a].i == matches[b].i {
            return matches[a].j > matches[b].j
        }
        return matches[a].i < matches[b].i
    })

    // Bước 3: LIS on j coordinates (Patience sorting)
    tails := []int{}
    for _, m := range matches {
        pos := sort.SearchInts(tails, m.j)
        if pos == len(tails) {
            tails = append(tails, m.j)
        } else {
            tails[pos] = m.j
        }
    }
    return len(tails)
}
03
Knapsack Problem
⭐⭐ Advanced

📌 Định nghĩa

Có n vật phẩm, mỗi vật có trọng lượng w[i] và giá trị v[i]. Túi có sức chứa W. Chọn các vật để tổng giá trị tối đa trong giới hạn trọng lượng. Biến thể: 0/1 Knapsack (mỗi vật dùng tối đa 1 lần), Unbounded (dùng không giới hạn), Fractional (cắt được — Greedy).

dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i])   nếu w ≥ weight[i]
dp[i][w] = dp[i-1][w]   nếu w < weight[i]
0/1 Knapsack
O(nW) / O(nW)
Mỗi item dùng 0 hoặc 1 lần
Space Optimized
O(nW) / O(W)
Rolling 1D array
Unbounded
O(nW) / O(W)
Item dùng nhiều lần
GO knapsack.go Advanced — 0/1 Knapsack đầy đủ 
package knapsack

type Item struct {
    Name   string
    Weight int
    Value  int
}

type KnapsackResult struct {
    MaxValue   int
    SelectedItems []Item
}

// Knapsack01: 0/1 Knapsack với item tracking
func Knapsack01(items []Item, capacity int) KnapsackResult {
    n := len(items)
    // dp[i][w] = max value dùng i items đầu với capacity w
    dp := make([][]int, n+1)
    for i := range dp {
        dp[i] = make([]int, capacity+1)
    }

    for i := 1; i <= n; i++ {
        item := items[i-1]
        for w := 0; w <= capacity; w++ {
            dp[i][w] = dp[i-1][w] // Không chọn item i
            if item.Weight <= w {
                take := dp[i-1][w-item.Weight] + item.Value
                if take > dp[i][w] {
                    dp[i][w] = take // Chọn item i
                }
            }
        }
    }

    // Backtrack để tìm các item được chọn
    selected := traceItems(dp, items, n, capacity)
    return KnapsackResult{
        MaxValue:      dp[n][capacity],
        SelectedItems: selected,
    }
}

func traceItems(dp [][]int, items []Item, n, w int) []Item {
    var selected []Item
    for i := n; i > 0; i-- {
        if dp[i][w] != dp[i-1][w] {
            selected = append(selected, items[i-1])
            w -= items[i-1].Weight
        }
    }
    return selected
}

// Knapsack01Opt: Space O(W) — rolling array
func Knapsack01Opt(items []Item, capacity int) int {
    dp := make([]int, capacity+1)
    for _, item := range items {
        // Duyệt NGƯỢC để tránh dùng item nhiều lần
        for w := capacity; w >= item.Weight; w-- {
            dp[w] = max(dp[w], dp[w-item.Weight]+item.Value)
        }
    }
    return dp[capacity]
}

// UnboundedKnapsack: Item dùng được nhiều lần
func UnboundedKnapsack(items []Item, capacity int) int {
    dp := make([]int, capacity+1)
    for w := 1; w <= capacity; w++ {
        for _, item := range items {
            if item.Weight <= w {
                // Duyệt XUÔI: cho phép dùng nhiều lần
                dp[w] = max(dp[w], dp[w-item.Weight]+item.Value)
            }
        }
    }
    return dp[capacity]
}
⚠️ Điểm quan trọng 0/1 Knapsack duyệt ngược từ W→weight[i] để tránh dùng item nhiều lần. Unbounded Knapsack duyệt xuôi từ weight[i]→W để cho phép dùng lại. Đây là điểm phân biệt chính giữa hai biến thể!
04
Matrix Chain Multiplication
⭐⭐⭐ Expert

📌 Định nghĩa

Cho n ma trận A₁, A₂, ..., Aₙ. Tìm thứ tự đặt ngoặc để tổng số phép nhân vô hướng là ít nhất. Ma trận Aᵢ có kích thước p[i-1] × p[i]. Đây là bài toán Interval DP kinh điển.

dp[i][j] = min over k in [i, j-1] of: dp[i][k] + dp[k+1][j] + p[i-1]*p[k]*p[j]
GO matrix_chain.go Expert — Matrix Chain + Parenthesization Trace 
package matrixchain

import (
    "fmt"
    "math"
    "strings"
)

type MCMResult struct {
    MinOps       int
    Parenthesis  string
    SplitTable   [][]int
}

// MatrixChain: O(n³) time, O(n²) space
// dims: kích thước [p0, p1, ..., pn] với matrix i có dims[i-1]xdims[i]
func MatrixChain(dims []int) MCMResult {
    n := len(dims) - 1 // số ma trận
    dp := make([][]int, n+1)
    split := make([][]int, n+1)
    for i := range dp {
        dp[i] = make([]int, n+1)
        split[i] = make([]int, n+1)
        for j := range dp[i] {
            dp[i][j] = math.MaxInt64
        }
        dp[i][i] = 0 // 1 ma trận: 0 phép nhân
    }

    // len = chiều dài chuỗi đang xét
    for length := 2; length <= n; length++ {
        for i := 1; i <= n-length+1; i++ {
            j := i + length - 1
            for k := i; k < j; k++ {
                // Chi phí: nhân A[i..k] với A[k+1..j]
                cost := dp[i][k] + dp[k+1][j] +
                        dims[i-1]*dims[k]*dims[j]
                if cost < dp[i][j] {
                    dp[i][j] = cost
                    split[i][j] = k
                }
            }
        }
    }

    paren := buildParen(split, 1, n)
    return MCMResult{
        MinOps:      dp[1][n],
        Parenthesis: paren,
        SplitTable:  split,
    }
}

func buildParen(split [][]int, i, j int) string {
    if i == j {
        return fmt.Sprintf("A%d", i)
    }
    k := split[i][j]
    left := buildParen(split, i, k)
    right := buildParen(split, k+1, j)
    return "(" + left + " × " + right + ")"
}

// MCM với Memo (Top-Down) — cho bài toán sparse
func MCMMemo(dims []int) int {
    n := len(dims) - 1
    memo := make([][]int, n+1)
    for i := range memo {
        memo[i] = make([]int, n+1)
        for j := range memo[i] { memo[i][j] = -1 }
        memo[i][i] = 0
    }
    var solve func(i, j int) int
    solve = func(i, j int) int {
        if memo[i][j] != -1 { return memo[i][j] }
        memo[i][j] = math.MaxInt64
        for k := i; k < j; k++ {
            cost := solve(i, k) + solve(k+1, j) +
                    dims[i-1]*dims[k]*dims[j]
            if cost < memo[i][j] { memo[i][j] = cost }
        }
        return memo[i][j]
    }
    _ = strings.Builder{}
    return solve(1, n)
}

Coin Change
⭐ Intermediate

📌 Định nghĩa

Cho một tập coin với các mệnh giá khác nhau và một số tiền amount. Tìm số coin ít nhất để tổng bằng amount (Coin Change I). Hoặc đếm tổng số cách đổi (Coin Change II). Đây là biến thể của Unbounded Knapsack.

dp[i] = min(dp[i], dp[i - coin] + 1)   với mỗi coin ≤ i

package coinchange

import "math"

// CoinChangeMin: Số coin ít nhất (LeetCode 322)
// dp[i] = số coin tối thiểu để tạo amount i
func CoinChangeMin(coins []int, amount int) int {
    dp := make([]int, amount+1)
    for i := 1; i <= amount; i++ {
        dp[i] = math.MaxInt32
    }
    for i := 1; i <= amount; i++ {
        for _, coin := range coins {
            if coin <= i && dp[i-coin] != math.MaxInt32 {
                dp[i] = min(dp[i], dp[i-coin]+1)
            }
        }
    }
    if dp[amount] == math.MaxInt32 { return -1 }
    return dp[amount]
}

// CoinChangeWays: Số cách đổi (LeetCode 518)
// dp[i] = số cách tạo amount i (order không quan trọng)
func CoinChangeWays(coins []int, amount int) int {
    dp := make([]int, amount+1)
    dp[0] = 1 // Base: 1 cách tạo amount 0
    // Duyệt coin trước → combination (không xét thứ tự)
    for _, coin := range coins {
        for i := coin; i <= amount; i++ {
            dp[i] += dp[i-coin]
        }
    }
    return dp[amount]
}

// CoinChangePermutations: Số cách đổi có xét thứ tự
// Duyệt amount trước → permutation
func CoinChangePerms(coins []int, amount int) int {
    dp := make([]int, amount+1)
    dp[0] = 1
    for i := 1; i <= amount; i++ {
        for _, coin := range coins {
            if i >= coin {
                dp[i] += dp[i-coin]
            }
        }
    }
    return dp[amount]
}

// TraceCoins: Tìm lại danh sách coin được chọn
func TraceCoins(coins []int, amount int) []int {
    dp := make([]int, amount+1)
    from := make([]int, amount+1)
    for i := 1; i <= amount; i++ {
        dp[i] = math.MaxInt32
    }
    for i := 1; i <= amount; i++ {
        for _, coin := range coins {
            if coin <= i && dp[i-coin]+1 < dp[i] {
                dp[i] = dp[i-coin] + 1
                from[i] = coin
            }
        }
    }
    var result []int
    for amount > 0 {
        result = append(result, from[amount])
        amount -= from[amount]
    }
    return result
}

House Robber
⭐ Intermediate

📌 Định nghĩa

Dãy nhà có giá trị nums[i]. Không được cướp hai nhà liền kề. Tìm tổng tiền tối đa. Biến thể: House Robber II (nhà xếp thành vòng tròn), House Robber III (nhà xếp thành cây nhị phân — Tree DP).

dp[i] = max(dp[i-1], dp[i-2] + nums[i])
GO house_robber.go Expert — I, II, III (Tree DP) 
package houserobber

// === HOUSE ROBBER I: Dãy thẳng ===
func Rob(nums []int) int {
    n := len(nums)
    if n == 0 { return 0 }
    if n == 1 { return nums[0] }
    // Space O(1): chỉ cần prev2, prev1
    prev2, prev1 := nums[0], max(nums[0], nums[1])
    for i := 2; i < n; i++ {
        curr := max(prev1, prev2+nums[i])
        prev2, prev1 = prev1, curr
    }
    return prev1
}

// === HOUSE ROBBER II: Vòng tròn ===
// Nhà đầu và cuối liền kề → chia thành 2 bài toán linear
func RobCircular(nums []int) int {
    n := len(nums)
    if n == 1 { return nums[0] }
    // Case 1: Không lấy nhà cuối [0..n-2]
    // Case 2: Không lấy nhà đầu [1..n-1]
    return max(robRange(nums, 0, n-2),
                robRange(nums, 1, n-1))
}

func robRange(nums []int, lo, hi int) int {
    prev2, prev1 := 0, 0
    for i := lo; i <= hi; i++ {
        prev2, prev1 = prev1, max(prev1, prev2+nums[i])
    }
    return prev1
}

// === HOUSE ROBBER III: Tree DP ===
// Nhà xếp thành cây, không cướp node cha và con cùng lúc
type TreeNode struct {
    Val   int
    Left  *TreeNode
    Right *TreeNode
}

// robTree trả về [maxWithRoot, maxWithoutRoot]
func robTree(node *TreeNode) (int, int) {
    if node == nil { return 0, 0 }

    leftWith, leftWithout := robTree(node.Left)
    rightWith, rightWithout := robTree(node.Right)

    // Lấy node này: không lấy con trực tiếp
    withRoot := node.Val + leftWithout + rightWithout
    // Không lấy node này: con có thể lấy hoặc không
    withoutRoot := max(leftWith, leftWithout) +
                   max(rightWith, rightWithout)

    return withRoot, withoutRoot
}

func RobTree(root *TreeNode) int {
    withRoot, withoutRoot := robTree(root)
    return max(withRoot, withoutRoot)
}
💡 Tree DP Pattern House Robber III là ví dụ kinh điển của Tree DP: mỗi node trả về tuple (lấy node này, không lấy node này), quyết định được đưa ra tại từng node dựa trên kết quả con.

Palindrome DP
⭐⭐⭐ Expert

📌 Định nghĩa

Các bài toán liên quan đến Palindrome: LPS (Longest Palindromic Subsequence), LPSubstring (Longest Palindromic Substring), Palindrome Partitioning (chia chuỗi thành ít phần palindrome nhất). Đây là Interval DP với dp[i][j] biểu diễn đoạn [i, j].

isPalin[i][j] = (s[i]==s[j]) && isPalin[i+1][j-1]
LPS: dp[i][j] = dp[i+1][j-1]+2 nếu s[i]==s[j], else max(dp[i+1][j], dp[i][j-1])

package palindrome

// === 1. Longest Palindromic Subsequence (LPS) ===
// dp[i][j] = độ dài LPS trong s[i..j]
func LPS(s string) int {
    n := len(s)
    dp := make([][]int, n)
    for i := range dp {
        dp[i] = make([]int, n)
        dp[i][i] = 1 // 1 ký tự là palindrome
    }
    for length := 2; length <= n; length++ {
        for i := 0; i <= n-length; i++ {
            j := i + length - 1
            if s[i] == s[j] && length == 2 {
                dp[i][j] = 2
            } else if s[i] == s[j] {
                dp[i][j] = dp[i+1][j-1] + 2
            } else {
                dp[i][j] = max(dp[i+1][j], dp[i][j-1])
            }
        }
    }
    return dp[0][n-1]
}

// === 2. Longest Palindromic Substring — Manacher's O(n) ===
// Thuật toán Manacher: tìm LPSubstring trong O(n) thay vì O(n²)
func LongestPalindromicSubstring(s string) string {
    // Transform: "abc" → "#a#b#c#" để xử lý even/odd đồng nhất
    t := transform(s)
    n := len(t)
    p := make([]int, n) // p[i] = radius của palindrome tại i
    c, r := 0, 0
    for i := 0; i < n; i++ {
        mirror := 2*c - i
        if i < r { p[i] = min(r-i, p[mirror]) }
        for i+p[i]+1 < n && i-p[i]-1 >= 0 &&
            t[i+p[i]+1] == t[i-p[i]-1] {
            p[i]++
        }
        if i+p[i] > r { c, r = i, i+p[i] }
    }
    maxLen, center := 0, 0
    for i, v := range p {
        if v > maxLen { maxLen, center = v, i }
    }
    start := (center - maxLen) / 2
    return s[start : start+maxLen]
}

func transform(s string) string {
    t := []byte{'#'}
    for _, c := range []byte(s) {
        t = append(t, c, '#')
    }
    return string(t)
}

// === 3. Palindrome Partitioning II (Min Cuts) ===
// dp[i] = số lần cắt tối thiểu cho s[0..i]
func MinCutPalindrome(s string) int {
    n := len(s)
    isPalin := make([][]bool, n)
    for i := range isPalin {
        isPalin[i] = make([]bool, n)
    }
    // Precompute isPalin[i][j]
    for i := n - 1; i >= 0; i-- {
        for j := i; j < n; j++ {
            isPalin[i][j] = s[i] == s[j] &&
                (j-i <= 2 || isPalin[i+1][j-1])
        }
    }
    dp := make([]int, n)
    for i := range dp { dp[i] = i } // max cuts = i
    for i := 1; i < n; i++ {
        if isPalin[0][i] { dp[i] = 0; continue }
        for j := 1; j <= i; j++ {
            if isPalin[j][i] {
                dp[i] = min(dp[i], dp[j-1]+1)
            }
        }
    }
    return dp[n-1]
}

package largestsquare

// === 1. Maximal Square (LeetCode 221) ===
func MaximalSquare(matrix [][]byte) int {
    if len(matrix) == 0 { return 0 }
    m, n := len(matrix), len(matrix[0])
    dp := make([][]int, m+1)
    for i := range dp { dp[i] = make([]int, n+1) }

    maxSide := 0
    for i := 1; i <= m; i++ {
        for j := 1; j <= n; j++ {
            if matrix[i-1][j-1] == '1' {
                dp[i][j] = min(
                    dp[i-1][j],
                    dp[i][j-1],
                    dp[i-1][j-1],
                ) + 1
                if dp[i][j] > maxSide { maxSide = dp[i][j] }
            }
        }
    }
    return maxSide * maxSide
}

// === 2. Maximal Rectangle (LeetCode 85) ===
// Kết hợp DP + Monotone Stack: O(mn) time
// Xem mỗi row là histogram, dùng Largest Rectangle in Histogram
func MaximalRectangle(matrix [][]byte) int {
    if len(matrix) == 0 { return 0 }
    m, n := len(matrix), len(matrix[0])
    heights := make([]int, n)
    maxArea := 0
    for i := 0; i < m; i++ {
        for j := 0; j < n; j++ {
            if matrix[i][j] == '1' {
                heights[j]++ // Tích lũy chiều cao
            } else {
                heights[j] = 0 // Reset khi gặp 0
            }
        }
        // Largest Rectangle in Histogram cho mỗi row
        area := largestInHistogram(heights)
        if area > maxArea { maxArea = area }
    }
    return maxArea
}

// Largest Rectangle in Histogram — Monotone Stack O(n)
func largestInHistogram(heights []int) int {
    stack := []int{-1}
    maxArea := 0
    for i, h := range heights {
        for stack[len(stack)-1] != -1 &&
            heights[stack[len(stack)-1]] >= h {
            top := stack[len(stack)-1]
            stack = stack[:len(stack)-1]
            width := i - stack[len(stack)-1] - 1
            maxArea = max(maxArea, heights[top]*width)
        }
        stack = append(stack, i)
    }
    n := len(heights)
    for stack[len(stack)-1] != -1 {
        top := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        width := n - stack[len(stack)-1] - 1
        maxArea = max(maxArea, heights[top]*width)
    }
    return maxArea
}

// === 3. Count Square Submatrices (LeetCode 1277) ===
// Đếm tổng số hình vuông (không chỉ lớn nhất)
func CountSquares(matrix [][]int) int {
    total := 0
    for i := range matrix {
        for j := range matrix[i] {
            if matrix[i][j] > 0 && i > 0 && j > 0 {
                matrix[i][j] = min(
                    matrix[i-1][j],
                    matrix[i][j-1],
                    matrix[i-1][j-1],
                ) + 1
            }
            total += matrix[i][j] // dp[i][j] = số hình vuông kết thúc tại đây
        }
    }
    return total
}
💡 Key Insight — min của 3 neighbor dp[i][j] = min(trên, trái, trên-trái) + 1 vì: hình vuông k×k tại (i,j) cần hình vuông (k-1)×(k-1) tại cả 3 góc còn lại. Nếu một trong 3 < k-1, không thể mở rộng thêm.
